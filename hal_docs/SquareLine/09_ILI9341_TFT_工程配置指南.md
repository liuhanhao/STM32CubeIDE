# SquareLine Studio — ILI9341 TFT 工程配置指南

> **适用硬件**: 2.4寸 ILI9341 TFT-LCD (240×320) + XPT2046 触摸  
> **适用平台**: ESP32 / STM32 / Arduino  
> **LVGL 版本**: 8.3.x（推荐）或 9.x  
> **通信接口**: SPI（4线）  
> **颜色格式**: RGB565（16-bit）

---

## 一、硬件参数确认

### 1.1 LCD 模块规格

根据模块技术文档，关键参数如下：

| 参数 | 值 | 说明 |
|------|-----|------|
| **LCD 控制器** | ILI9341V | 支持 SPI/并口，本配置使用 SPI |
| **分辨率** | 240×320 | 宽×高，竖屏模式 |
| **颜色深度** | 16-bit (RGB565) | SquareLine 和代码必须一致 |
| **接口类型** | 4 线 SPI | SCK, MOSI, MISO, CS, DC, RST |
| **触摸控制器** | XPT2046 (TSC2046) | SPI 接口，4线电阻式触摸 |

### 1.2 平台差异对照

| 项目 | ESP32 | STM32F103 | Arduino Uno |
|------|-------|-----------|-------------|
| **主频** | 240MHz | 72MHz | 16MHz |
| **RAM** | 520KB | 20KB | 2KB |
| **推荐 SPI 频率** | 40MHz | 18-36MHz | 8MHz |
| **帧缓冲策略** | 1/5 ~ 1/2 屏 | 1/10 ~ 1/20 屏 | 单行缓冲 |
| **DMA 支持** | ✅ | ✅ | ❌ |

## 三、屏幕方向配置

### 3.1 竖屏模式（240×320，默认）

- **SquareLine 分辨率**: 240×320
- **LCD 驱动 0x36 命令**: `0x48`（MX=1, BGR=1）
- **适用场景**: 手表、温度计、垂直显示器

### 3.2 横屏模式（320×240）

- **SquareLine 分辨率**: 320×240
- **LCD 驱动 0x36 命令**: `0x28`（MV=1, BGR=1）
- **适用场景**: HMI 控制面板、仪表盘

> ⚠️ **SquareLine 分辨率必须与 LCD 驱动方向匹配**，否则会显示错位！

## 四、平台集成配置

### 4.1 ESP32 平台（PlatformIO）

#### **platformio.ini 配置**

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino

lib_deps = 
    lvgl/lvgl@^8.3.11           # LVGL 库
    bodmer/TFT_eSPI@^2.5.43     # LCD 驱动（或用 lvgl_esp32_drivers）
    
build_flags = 
    -D LV_CONF_INCLUDE_SIMPLE
    -D USER_SETUP_LOADED=1
    -I src
```

#### **lv_conf.h 关键配置**

```c
/* ESP32 推荐配置 */
#define LV_COLOR_DEPTH 16
#define LV_COLOR_16_SWAP 0              // 根据硬件调整

#define LV_HOR_RES_MAX 240              // 竖屏宽度
#define LV_VER_RES_MAX 320              // 竖屏高度

#define LV_DISP_DEF_REFR_PERIOD 10      // 10ms（100fps，ESP32 性能充足）
#define LV_MEM_SIZE (64U * 1024U)       // 64KB 内存池（ESP32 RAM 充足）

#define LV_USE_PERF_MONITOR 1
#define LV_USE_MEM_MONITOR 1
```

#### **主程序集成（main.cpp）**

```cpp
#include <Arduino.h>
#include <lvgl.h>
#include <TFT_eSPI.h>
#include "ui/ui.h"                      // SquareLine 导出

TFT_eSPI tft = TFT_eSPI();
static lv_disp_draw_buf_t draw_buf;
static lv_color_t buf[240 * 20];        // 1/16 屏缓冲

void my_disp_flush(lv_disp_drv_t *disp, const lv_area_t *area, lv_color_t *color_p) {
    uint32_t w = (area->x2 - area->x1 + 1);
    uint32_t h = (area->y2 - area->y1 + 1);
    tft.startWrite();
    tft.setAddrWindow(area->x1, area->y1, w, h);
    tft.pushColors((uint16_t *)&color_p->full, w * h, true);
    tft.endWrite();
    lv_disp_flush_ready(disp);
}

void setup() {
    Serial.begin(115200);
    
    // LCD 初始化
    tft.begin();
    tft.setRotation(0);                 // 0=竖屏，1=横屏
    
    // LVGL 初始化
    lv_init();
    lv_disp_draw_buf_init(&draw_buf, buf, NULL, 240 * 20);
    
    static lv_disp_drv_t disp_drv;
    lv_disp_drv_init(&disp_drv);
    disp_drv.hor_res = 240;
    disp_drv.ver_res = 320;
    disp_drv.flush_cb = my_disp_flush;
    disp_drv.draw_buf = &draw_buf;
    lv_disp_drv_register(&disp_drv);
    
    // 加载 SquareLine UI
    ui_init();                          // ✨ 这里！
}

void loop() {
    lv_timer_handler();                 // LVGL 任务处理
    delay(5);
}
```

### 4.2 STM32 平台（CubeIDE）

#### **lv_conf.h 关键配置**

```c
/* STM32F103 推荐配置（内存受限） */
#define LV_COLOR_DEPTH 16
#define LV_COLOR_16_SWAP 0

#define LV_HOR_RES_MAX 240
#define LV_VER_RES_MAX 320

#define LV_DISP_DEF_REFR_PERIOD 33      // 33ms（30fps，节省 CPU）
#define LV_MEM_SIZE (12U * 1024U)       // 12KB 内存池（RAM 有限）

#define LV_USE_PERF_MONITOR 0           // 节省内存可关闭
```

#### **主程序集成（main.c）**

```c
/* USER CODE BEGIN Includes */
#include "lvgl.h"
#include "lv_port_disp.h"              // 显示驱动移植
#include "lv_port_indev.h"             // 触摸驱动移植
#include "ui/ui.h"                     // SquareLine 导出 ✨
/* USER CODE END Includes */

int main(void) {
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_SPI1_Init();
    
    /* USER CODE BEGIN 2 */
    lv_init();                         // LVGL 初始化
    lv_port_disp_init();               // 显示驱动初始化
    lv_port_indev_init();              // 触摸驱动初始化
    ui_init();                         // SquareLine UI 初始化 ✨
    /* USER CODE END 2 */
    
    while (1) {
        lv_task_handler();             // LVGL 任务处理
        HAL_Delay(5);
    }
}
```

#### **SysTick 配置（stm32f1xx_it.c）**

```c
void SysTick_Handler(void) {
    HAL_IncTick();
    lv_tick_inc(1);                    // LVGL 1ms 时基 ✨
}
```

## 五、导出与集成流程

### 5.1 设置导出路径

1. **File** → **Project Settings**
2. 设置 **UI Files Export Path**:
   - ESP32: `/your_platformio_project/src/ui`
   - STM32: `/your_stm32_project/Core/Src/ui`

### 5.2 导出文件

**File** → **Export** → **Export UI Files**

生成的文件结构：

```
ui/
├── ui.c                          ← 主入口
├── ui.h                          ← 头文件
├── ui_Screen_*.c                 ← 各屏幕实现
├── ui_Screen_*.h
├── ui_events.c                   ← 事件回调（手动编辑）
├── ui_events.h
├── ui_helpers.c
├── ui_helpers.h
└── images/                       ← 图片资源（如有）
    └── ui_img_*.c
```

### 5.3 添加到工程

- **ESP32**: 将 `ui/` 文件夹复制到 `src/` 目录，PlatformIO 会自动编译
- **STM32**: 复制到工程后，右键 `ui/` → **Add/Remove Include Path**

## 六、常见问题排查

### 6.1 显示花屏/颜色错误

**可能原因**:
1. 颜色深度不匹配（SquareLine 16bit，但代码设为 32bit）
2. RGB/BGR 顺序错误
3. SPI 字节序问题

**解决方案**:
```c
// 尝试调整这两个参数
#define LV_COLOR_16_SWAP 0              // 改为 1 试试
#define LV_COLOR_ORDER_SWAP 1           // 如果有这个宏
```

### 6.2 显示内容旋转/错位

**原因**: SquareLine 分辨率与 LCD 驱动方向不匹配

**解决**:
| SquareLine | LCD setRotation() | LCD 0x36 命令 |
|-----------|-------------------|--------------|
| 240×320 | `0` | `0x48` |
| 320×240 | `1` | `0x28` |

### 6.3 触摸坐标不准

**原因**: 触摸未校准

**解决**:
1. 运行校准程序（参考硬件文档 `CALIBRATION_MODES.md`）
2. 修改 `lv_port_indev.c` 中的坐标映射函数

### 6.4 内存溢出（STM32）

**现象**: 程序运行一段时间后卡死

**解决**:
```c
// 减小 LVGL 内存池
#define LV_MEM_SIZE (8U * 1024U)        // 从 12KB 减到 8KB

// 减小显示缓冲
static lv_color_t buf[240 * 10];        // 从 1/10 屏减到单行
```

### 6.5 刷新卡顿

**优化方案**:

| 平台 | 优化方法 |
|------|---------|
| **ESP32** | 1. 提高 SPI 到 40MHz<br>2. 使用 DMA<br>3. 增大缓冲区到 1/5 屏 |
| **STM32** | 1. 使用 DMA<br>2. SPI 设为 36MHz<br>3. 简化 UI 动画 |
