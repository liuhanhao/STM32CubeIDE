# STM32 外设与通信协议速记笔记

> 适用芯片：STM32F103C8T6（Blue Pill）  
> 整理目标：快速理解各外设用途、工作原理与常用组合

---

## 一、中断

### 1. 外部中断（EXTI）
- **触发来源**：外部硬件引脚电平变化（上升沿 / 下降沿 / 双边沿）
- **典型用途**：按键检测、传感器信号触发

### 2. 内部中断
- **触发来源**：单片机内部模块（如定时器溢出）
- **典型用途**：定时任务、周期性处理

---

## 二、TIM 定时器

| 模式 | 说明 | 典型用途 |
|------|------|----------|
| 普通定时中断 | 按设定周期触发中断 | 计时、周期任务调度 |
| PWM 输出 | 输出可调占空比方波 | 电机调速、LED 调光、舵机控制 |
| 输入捕获（单路） | 测量单路信号高 / 低电平时间 | 测频率、测PWM占空比 |
| 输入捕获（编码器模式） | 双路正交信号硬件自动判向 | 旋转编码器、电机测速、里程计数 |

### PWM 关键参数
- **频率（Hz）**：控制运行平顺度、LED 闪烁感
- **占空比（%）**：控制转速、亮度、功率大小

### 输入捕获 vs 编码器模式对比

| 特性 | 输入捕获 | 编码器模式 |
|------|----------|-----------|
| 自动识别方向 | ❌ 需自己写逻辑 | ✅ 硬件自动判断 |
| 自动计数增减 | ❌ | ✅ |
| 接线通道 | 单路 | TIM_CH1 + CH2 接 A/B 相 |
| 首选场景 | 测频率 / 占空比 | 旋转编码器 / 电机测速 |

---

## 三、ADC（模拟 → 数字）

- **作用**：将连续模拟电压转换为单片机可识别的数字数值
- **最大测量电压**：3.3V
- **计算公式**：

$$实际电压 = \frac{ADC读取数值}{4095} \times 3.3$$

| 模式 | 说明 |
|------|------|
| 单通道 | 固定采集一路模拟电压，代码简单 |
| 多通道 | 硬件轮流扫描多路，搭配 DMA 效率最高 |

---

## 四、DMA（数据搬运）

- **作用**：硬件自动搬运数据，**不占用 CPU 资源**，解放单片机
- **三种搬运方向**：

| 方向 | 说明 |
|------|------|
| 外设 → 内存 | 收数据（如 ADC 采集结果存入数组） |
| 内存 → 外设 | 发数据（如串口批量发送缓冲区） |
| 内存 → 内存 | 纯拷贝（如数据备份） |

### 常用组合搭配

| 组合 | 搬运方向 | 用途 |
|------|----------|------|
| ADC 多通道 + DMA | 外设 → 内存 | 持续采集多路模拟信号 |
| 串口大量收发 + DMA | 双向 | 高速串口通信 |
| LCD 刷屏 + DMA | 内存 → 外设 | 快速刷新显示缓冲区 |
| 内存数据备份 | 内存 → 内存 | 数据搬移拷贝 |

---

## 五、DAC（数字 → 模拟）

- **作用**：输出真实稳定的模拟电压，直接可调
- **与 PWM 的区别**：

| 特性 | DAC | PWM |
|------|-----|-----|
| 输出方式 | 直接输出平稳电压 | 高低电平快速通断，靠平均值调压 |
| 调压类型 | 直接调压 | 间接调压 |
| 波形质量 | 平滑连续 | 需滤波才能平滑 |
| 典型用途 | 音频输出、精准电压控制 | 电机调速、LED 调光 |

---

## 六、存储器：Flash 与 SRAM

| 特性 | Flash | SRAM |
|------|-------|------|
| 类比 | 硬盘 | 运行内存 |
| 存储内容 | 程序代码、常量 | 变量、临时数据 |
| 断电保留 | ✅ 保留 | ❌ 清空 |
| 读取速度 | 快 | 最快 |
| 擦写速度 | 慢（有次数限制） | 任意读写 |

## 七、BKP 备份寄存器

### 基本概念
- **全称**：Backup Registers，备份寄存器
- **位置**：属于后备区域（Backup Domain），与 RTC 共用供电域
- **供电**：由 VBAT 引脚供电（接纽扣电池），主电源断电后数据依然保留
- **容量**：STM32F103C8T6 共 **10 个 16 位寄存器**（BKP_DR1 ~ BKP_DR10），共 20 字节

### 核心特性

| 特性 | 说明 |
|------|------|
| 断电保持 | 只要 VBAT 有电（电池），数据永久保留 |
| 掉电恢复 | 主电源重新上电后可读回之前存的值 |
| 与 RTC 联动 | 同在后备域，RTC 校准值也存在 BKP 里 |
| 写保护 | 写入前必须先解除后备区域写保护（PWR_CR 的 DBP 位） |
| 复位不清零 | 系统复位、看门狗复位均不清除 BKP 数据 |
| 侵入检测 | TAMPER 引脚可配置为入侵检测，触发后自动清除所有 BKP 数据 |

### 与其他存储的对比

| 存储类型 | 断电保持 | 容量 | 擦写次数 | 典型用途 |
|----------|----------|------|----------|----------|
| BKP 寄存器 | ✅（需 VBAT） | 20 字节 | 无限制 | 少量关键参数、上电标志 |
| Flash | ✅ 无需电池 | 64KB | ~10万次 | 程序代码、大量配置 |
| SRAM | ❌ 断电清零 | 20KB | 无限制 | 运行变量 |

### 典型用途
- **上电标志**：写入特定值（如 `0xA5A5`），下次上电读取判断是否首次启动
- **RTC 校准**：存储 RTC 时钟校准参数，避免每次重新校准
- **参数保存**：少量需要掉电保持的配置参数（设备 ID、运行模式等）
- **入侵记录**：配合 TAMPER 引脚检测设备是否被非法拆机

### 使用注意
1. 必须先开启 PWR 和 BKP 时钟（`__HAL_RCC_PWR_CLK_ENABLE()`、`__HAL_RCC_BKP_CLK_ENABLE()`）
2. 写入前解除写保护：`HAL_PWR_EnableBkUpAccess()`
3. VBAT 悬空时数据不保证保留，生产环境务必接电池或接 VDD

```c
/* 写入 BKP */
HAL_PWR_EnableBkUpAccess();                    // 解除写保护
HAL_RTCEx_BKUPWrite(&hrtc, RTC_BKP_DR1, 0xA5A5); // 写入数据

/* 读取 BKP */
uint32_t val = HAL_RTCEx_BKUPRead(&hrtc, RTC_BKP_DR1);
```

### BKP 与 RTC 配合使用

BKP 和 RTC 同属**后备域（Backup Domain）**，共用 VBAT 供电，天然是黄金搭档。

#### 为什么要配合使用

RTC 断电后靠电池继续走时，但每次主电源上电时代码都会重新初始化 RTC（设置时间），导致时间被重置。解决方案：**用 BKP 存一个标志位**，上电时先检查标志，判断是否首次启动，从而决定要不要重新设置时间。

```
首次上电 → BKP 没有标志 → 初始化 RTC，设置初始时间 → 写入 BKP 标志
再次上电 → BKP 有标志   → 跳过 RTC 时间设置 → RTC 继续走之前的时间
```

#### 标准初始化流程

```c
#define RTC_INIT_FLAG  0xA5A5   // 自定义标志值

void RTC_Init_WithBKP(void) {
    // 1. 使能后备域时钟和访问权限
    __HAL_RCC_PWR_CLK_ENABLE();
    __HAL_RCC_BKP_CLK_ENABLE();
    HAL_PWR_EnableBkUpAccess();

    // 2. 检查 BKP 标志，判断是否首次上电
    if (HAL_RTCEx_BKUPRead(&hrtc, RTC_BKP_DR1) != RTC_INIT_FLAG) {
        // 首次上电：初始化 RTC 并设置时间
        HAL_RTC_Init(&hrtc);

        RTC_TimeTypeDef sTime = {0};
        RTC_DateTypeDef sDate = {0};
        sTime.Hours = 0; sTime.Minutes = 0; sTime.Seconds = 0;
        sDate.Year = 26; sDate.Month = 1; sDate.Date = 1;
        HAL_RTC_SetTime(&hrtc, &sTime, RTC_FORMAT_BIN);
        HAL_RTC_SetDate(&hrtc, &sDate, RTC_FORMAT_BIN);

        // 写入标志，下次上电不再重置时间
        HAL_RTCEx_BKUPWrite(&hrtc, RTC_BKP_DR1, RTC_INIT_FLAG);
    }
    // 非首次上电：直接跳过，RTC 继续走之前的时间
}
```

#### 扩展：BKP 存更多 RTC 相关数据

| BKP 寄存器 | 典型存储内容 |
|-----------|-------------|
| BKP_DR1 | 初始化标志（`0xA5A5`）|
| BKP_DR2 | RTC 校准值 |
| BKP_DR3 | 设备运行次数 / 开机计数 |
| BKP_DR4~DR10 | 用户自定义掉电保留参数 |

#### 关键注意事项
- RTC 时钟源推荐使用 **LSE（32.768KHz 外部晶振）**，精度高；LSI 为内部 RC，误差较大
- VBAT 悬空时，BKP 标志会丢失，下次上电会误判为首次启动并重置时间
- 读取 RTC 时间时，**必须先读 Time 再读 Date**，顺序不能反（HAL 库内部有锁存机制）

## 八、通信协议

### 7.1 UART 与 USART

| 特性 | UART | USART |
|------|------|-------|
| 通信方式 | 仅异步 | 异步 + 同步 |
| 引脚 | RX、TX | RX、TX（同步模式额外加 SCLK） |
| 同步时钟 | ❌ | ✅ SCLK 统一节拍 |
| 日常使用 | 全部当普通异步串口，接线用法一致 | |

> **同步 SCLK 的作用**：统一传输节拍，不靠波特率猜时间，速度更快、数据更稳，适合大数据量传输。

---

### 7.2 I2C

- **引脚**：SDA（数据）+ SCL（时钟），共 2 根线
- **特点**：支持多设备挂载，低速外设首选
- **典型外设**：OLED、温湿度传感器、EEPROM
- **示例**：LCD1602 原生为并行通信，加装 I2C 转接板后节省大量 IO 口

#### I2C 时序操作

**指定地址写**

对于指定设备（Slave Address），在指定地址（Reg Address）下，写入指定数据（Data）。

```
主机发送：START → Slave Address + W → ACK → Reg Address → ACK → Data → ACK → STOP
```

#### 软件 I2C vs 硬件 I2C

| 对比项 | 软件 I2C | 硬件 I2C |
|--------|----------|----------|
| 实现方式 | 用 GPIO 手动模拟时序（位拼协议） | 使用芯片内置 I2C 外设模块 |
| 引脚限制 | ✅ 任意 GPIO 均可使用 | ❌ 只能用固定的 I2C 引脚（如 PB6/PB7） |
| 移植性 | ✅ 换芯片只改 GPIO 操作 | ❌ 依赖芯片 HAL 库，换平台需重写 |
| CPU 占用 | ❌ 高，时序全靠 CPU 循环等待 | ✅ 低，硬件自动处理时序，可配合 DMA/中断 |
| 速度 | ❌ 受限于 CPU 执行速度，较慢 | ✅ 硬件级速度，支持标准/快速/超快速模式 |
| 稳定性 | ❌ 受中断打断影响，时序可能抖动 | ✅ 硬件保证时序精准，不受中断影响 |
| 调试难度 | ✅ 代码透明，出错容易定位 | ❌ HAL 库封装深，出错不好排查 |
| 代码复杂度 | ❌ 需要自己实现完整时序逻辑 | ✅ 调用 HAL 函数即可，代码简洁 |

**选用建议：**
- 引脚不够用 / 需要接多个 I2C 设备在不同引脚 → 用**软件 I2C**
- 追求性能、稳定性、低 CPU 占用 → 用**硬件 I2C**
- 学习阶段理解协议原理 → 先写一遍**软件 I2C**，再用硬件 I2C

---

### 7.3 SPI

- **引脚**：MOSI、MISO、SCK、CS，共 4 根线
- **特点**：速度最快，全双工，适合高速外设
- **典型外设**：Flash、SD 卡、高速 DAC/ADC

#### 软件 SPI vs 硬件 SPI

| 对比项 | 软件 SPI | 硬件 SPI |
|--------|----------|----------|
| 实现方式 | 用 GPIO 手动翻转时钟、数据线模拟时序 | 使用芯片内置 SPI 外设模块 |
| 引脚限制 | ✅ 任意 GPIO 均可使用 | ❌ 只能用固定的 SPI 引脚（如 PA5/PA6/PA7） |
| 移植性 | ✅ 换芯片只改 GPIO 操作 | ❌ 依赖芯片 HAL 库，换平台需重写 |
| CPU 占用 | ❌ 高，逐位翻转靠 CPU 循环 | ✅ 低，硬件移位寄存器自动完成，可配合 DMA |
| 速度 | ❌ 慢，受限于 CPU 执行频率 | ✅ 快，可达几十 MHz，接近总线极限 |
| 稳定性 | ❌ 被中断打断会导致时序错误 | ✅ 硬件保证时序，不受中断影响 |
| 多设备支持 | ✅ CS 引脚随意分配，扩展方便 | ⚠️ CS 也可用任意 GPIO，但 SCK/MOSI/MISO 共用固定引脚 |
| 调试难度 | ✅ 代码透明，逻辑清晰易排查 | ❌ HAL 封装深，出错时不好定位 |
| 代码复杂度 | ❌ 需手动实现时钟极性（CPOL）、相位（CPHA）等细节 | ✅ HAL 函数一行搞定收发 |

**选用建议：**
- 引脚资源紧张 / 需要在非标准引脚上挂 SPI 设备 → 用**软件 SPI**
- 高速传输（如大容量 Flash、LCD 刷屏）→ 必须用**硬件 SPI**
- 同时挂多个 SPI 设备 → 硬件 SPI 共享 SCK/MOSI/MISO，CS 各自独立 GPIO
- 学习 SPI 协议原理 → 先实现一遍**软件 SPI**，理解 CPOL/CPHA 四种模式

---

### 7.4 单总线（DHT11）

- **特点**：自定义协议，**单线双向**通信
- **用途**：温湿度采集（DHT11 / DHT22）
- 一根数据线完成所有收发，协议有严格时序要求

---

### 7.5 通信协议横向对比

| 协议 | 线数 | 速度 | 特点 | 典型用途 |
|------|------|------|------|----------|
| UART | 2（RX/TX） | 低~中 | 最常用，调试首选 | 串口调试、模块通信 |
| I2C | 2（SDA/SCL） | 低~中 | 多设备共享总线 | 传感器、显示屏 |
| SPI | 4 | 最快 | 全双工，点对点 | Flash、SD卡、屏幕 |
| 单总线 | 1 | 低 | 时序严格，接线极简 | DHT11 温湿度 |

## 十一、综合组合速查

| 场景 | 推荐组合 |
|------|----------|
| 持续采集多路模拟信号 | ADC 多通道 + DMA（外设→内存） |
