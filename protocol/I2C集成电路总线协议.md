# I²C 集成电路总线协议

## 概述

I²C（Inter-Integrated Circuit，集成电路总线）是由飞利浦公司开发的一种**同步、半双工**串行通讯协议。它只需两根线就能连接多个设备，通过地址区分不同从机，非常节省引脚资源。

- 通讯方式：同步、半双工（同一时刻只能单向传输）
- 所需引脚：SDA（数据线）、SCL（时钟线）
- 典型速度：标准模式 100 kHz，快速模式 400 kHz，高速模式 3.4 MHz
- 支持：多主多从，通过 7 位或 10 位地址区分设备

---

## 硬件连接

```
MCU（主机）          从机1（EEPROM）    从机2（温度传感器）
   SDA ─────┬──────────── SDA ──────────── SDA
   SCL ─────┼──────────── SCL ──────────── SCL
            │
           上拉电阻（4.7kΩ）
            │
           VCC（3.3V 或 5V）
```

> SDA 和 SCL 都需要接上拉电阻到 VCC，总线默认高电平。这是 I²C 的开漏（Open-Drain）特性要求。

## 关键信号定义

### 起始条件（START）
SCL 高电平期间，SDA 从高变低

```
SCL: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
SDA: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾_
                          ↑ START
```

### 停止条件（STOP）
SCL 高电平期间，SDA 从低变高

```
SCL: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
SDA: ___________________‾‾
                        ↑ STOP
```

### 数据传输
SCL 高电平期间 SDA 必须稳定，SCL 低电平期间 SDA 才能变化

```
SCL: _‾‾‾‾_‾‾‾‾_‾‾‾‾_
SDA: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾  （稳定的数据位）
         ↑采样  ↑采样  ↑采样
```

### 应答位（ACK/NACK）
每传输 8 位数据后，接收方需发送 1 位应答：
- **ACK（0）**：拉低 SDA，表示接收成功
- **NACK（1）**：释放 SDA（高电平），表示接收失败或结束

## 完整帧结构

```
┌───────┬──────────────┬─────┬──────────────────┬─────┬────────┐
│ START │  从机地址(7位) │ R/W │  数据字节(8位)    │ ACK │  STOP  │
└───────┴──────────────┴─────┴──────────────────┴─────┴────────┘
```

| 字段 | 位数 | 说明 |
|------|------|------|
| START | - | 起始条件 |
| 从机地址 | 7位 | 目标设备地址（0x00～0x7F） |
| R/W | 1位 | 0=写，1=读 |
| ACK | 1位 | 接收方应答 |
| 数据 | 8位 | 实际传输的数据 |
| STOP | - | 停止条件 |

## 通讯过程详解

### 写操作流程

```
主机                              从机（地址 0x50）
  │                                    │
  │── START ──────────────────────────→│
  │── 地址 0x50 + W(0) ───────────────→│
  │←─ ACK ────────────────────────────│ （从机确认）
  │── 寄存器地址 0x00 ────────────────→│
  │←─ ACK ────────────────────────────│
  │── 数据 0xAB ──────────────────────→│
  │←─ ACK ────────────────────────────│
  │── STOP ───────────────────────────→│
```

### 读操作流程（先写寄存器地址，再读数据）

```
主机                              从机（地址 0x50）
  │                                    │
  │── START ──────────────────────────→│
  │── 地址 0x50 + W(0) ───────────────→│  （先写，指定要读的寄存器）
  │←─ ACK ────────────────────────────│
  │── 寄存器地址 0x00 ────────────────→│
  │←─ ACK ────────────────────────────│
  │                                    │
  │── 重复START（ReSTART）────────────→│  （不发STOP，直接重新开始）
  │── 地址 0x50 + R(1) ───────────────→│  （切换为读模式）
  │←─ ACK ────────────────────────────│
  │←─ 数据 0xAB ──────────────────────│  （从机发送数据）
  │── NACK ───────────────────────────→│  （主机不再需要数据，发NACK）
  │── STOP ───────────────────────────→│
```

## 完整通讯示例

**场景：** 单片机通过 I²C 读写 AT24C02 EEPROM（地址 0x50）

### 写入一个字节到 EEPROM

向地址 0x10 写入数据 0x55：

```
时序：
START → [0xA0] → ACK → [0x10] → ACK → [0x55] → ACK → STOP
         ↑                ↑               ↑
    设备地址+W         内存地址          数据

注：0xA0 = 0b10100000 = 地址0x50左移1位 + W(0)
```

```c
// 向 AT24C02 写入一个字节
void EEPROM_WriteByte(uint8_t memAddr, uint8_t data) {
    I2C_Start();                     // 发送起始条件
    I2C_SendByte(0xA0);              // 发送设备地址 + 写标志（0xA0）
    I2C_WaitACK();                   // 等待从机应答
    I2C_SendByte(memAddr);           // 发送内存地址
    I2C_WaitACK();
    I2C_SendByte(data);              // 发送数据
    I2C_WaitACK();
    I2C_Stop();                      // 发送停止条件
    delay_ms(5);                     // 等待 EEPROM 内部写入完成
}
```

### 从 EEPROM 读取一个字节

从地址 0x10 读取数据：

```
时序：
START → [0xA0] → ACK → [0x10] → ACK → ReSTART → [0xA1] → ACK → [数据] → NACK → STOP
         ↑                ↑                          ↑
    设备地址+W         内存地址                  设备地址+R

注：0xA1 = 0b10100001 = 地址0x50左移1位 + R(1)
```

```c
// 从 AT24C02 读取一个字节
uint8_t EEPROM_ReadByte(uint8_t memAddr) {
    uint8_t data;

    I2C_Start();                     // 起始条件
    I2C_SendByte(0xA0);              // 设备地址 + 写（先写地址）
    I2C_WaitACK();
    I2C_SendByte(memAddr);           // 发送要读取的内存地址
    I2C_WaitACK();

    I2C_Start();                     // 重复起始条件（ReSTART）
    I2C_SendByte(0xA1);              // 设备地址 + 读
    I2C_WaitACK();

    data = I2C_ReadByte();           // 读取数据
    I2C_SendNACK();                  // 发送 NACK，告知从机不再读取
    I2C_Stop();                      // 停止条件

    return data;
}
```

## I²C 地址说明

7 位地址模式下，地址范围 0x00～0x7F，但部分地址被保留：

| 地址范围 | 用途 |
|----------|------|
| 0x00 | 广播地址（所有设备响应） |
| 0x01～0x07 | 保留 |
| 0x08～0x77 | 可用设备地址 |
| 0x78～0x7F | 保留（10位地址扩展用） |

常见设备地址示例：

| 设备 | 典型地址 |
|------|----------|
| AT24C02 EEPROM | 0x50～0x57（由A0/A1/A2引脚决定） |
| DS3231 RTC | 0x68 |
| MPU6050 陀螺仪 | 0x68 或 0x69 |
| SHT30 温湿度 | 0x44 或 0x45 |
| BMP280 气压 | 0x76 或 0x77 |

## 多设备总线示例

**场景：** 同一 I²C 总线上挂载 RTC（0x68）和温湿度传感器（0x44）

```
主机读取 RTC 时间：
START → [0xD0] → ACK → [0x00] → ACK → ReSTART → [0xD1] → ACK → [秒] → [分] → [时] → NACK → STOP

主机读取温度：
START → [0x88] → ACK → [0x2C06] → ACK → ReSTART → [0x89] → ACK → [温度高] → [温度低] → NACK → STOP
```

两次通讯完全独立，通过不同地址区分设备，无需额外引脚。

## 常见问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 总线一直低电平 | 上拉电阻缺失或过大 | 检查上拉电阻（推荐4.7kΩ） |
| 无 ACK 响应 | 地址错误或设备未上电 | 确认设备地址和供电 |
| 数据错误 | 速度过快 | 降低 SCL 频率 |
| 总线死锁 | 通讯中断导致从机占用SDA | 手动产生9个SCK脉冲复位从机 |

## 典型应用

- EEPROM 存储（AT24C系列）
- RTC 实时时钟（DS3231、PCF8563）
- 温湿度传感器（SHT30、HDC1080）
- IMU 惯性测量（MPU6050、ICM42688）
- OLED 显示屏（SSD1306）
