# 触摸功能故障排查指南

> **适用场景**: 触摸无响应或坐标异常  
> **最后更新**: 2026-06-11

---

## 快速诊断

### 症状1: 触摸完全无反应

**最可能原因**: 触摸VCC未供电 ⚡

**快速检查**:
```
用万用表测量触摸排针：
- VCC针: 应该是3.3V ✓
- GND针: 应该是0V ✓
- T_IRQ针: 无触摸时3.3V，触摸时0V ✓
```

**解决方法**:
1. 确认触摸VCC接到3.3V（不是GPIO！）
2. 确认触摸GND接到GND
3. 如果LCD和触摸是一体模块，VCC/GND可能共用

---

### 症状2: 触摸有反应但坐标不准

**可能原因**: 未校准或校准不准确

**解决方法**:
1. 重新执行四点校准（点击CAL按钮）
2. 校准时**精确点击**十字准心中心
3. 如果自动失败，会重新开始

---

### 症状3: IRQ引脚无变化

**可能原因**: 接线错误

**诊断方法**:
```c
// 在main.c的Touch_Test()中添加调试
while(1) {
    uint8_t irq_state = HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_3);
    sprintf(buf, "IRQ: %d", irq_state);
    LCD_ShowString(10, 100, 16, buf, 0);
    HAL_Delay(100);
}
```

**预期结果**:
- 无触摸时: IRQ = 1（高电平）
- 触摸时: IRQ = 0（低电平）

如果始终是1或0，说明引脚接错

## 接线问题排查

### 1. 触摸排针识别

触摸屏模块通常有以下几种接口：

**形式A: LCD和触摸合并排针**
```
[VCC][GND][CS][RST][DC][MOSI][SCK][LED][MISO][T_IRQ][T_DO][T_DIN][T_CS][T_CLK]
```

**形式B: 独立触摸排针（5针）**
```
触摸排针: [T_IRQ][T_DO][T_DIN][T_CS][T_CLK]
（VCC/GND与LCD共用）
```

**形式C: 独立触摸排针（7针，含供电）**
```
触摸排针: [VCC][GND][T_IRQ][T_DO][T_DIN][T_CS][T_CLK]
```

### 2. 确认引脚顺序

**方法1: 看丝印**
- 模块PCB上通常有标注
- 找"T_IRQ"或"TPENIRQ"标记

**方法2: 看原理图**
- 查看资料包中的"sch模块原理图.pdf"

**方法3: 万用表测试**（推荐）
```
黑表笔接GND，红表笔依次测量触摸排针：

预期结果：
- VCC针: 3.3V（不变）
- GND针: 0V（不变）
- T_IRQ针: 3.3V → 0V（触摸时变化） ← 找到这个就对了
- 其他针: 不确定
```

### 3. 常见接线错误

**错误1: 排针方向接反**
```
正确: 模块正面朝上，从左到右数
错误: 模块背面朝上，顺序就反了
```

**错误2: VCC接到GPIO了**
```
如果触摸排针是:
[VCC][GND][T_IRQ][T_DO][T_DIN][T_CS][T_CLK]

错误接法:
VCC → PA3  ✗ (VCC应该接3.3V供电!)
GND → PA4  ✗
...

正确接法:
VCC → 3.3V  ✓
GND → GND   ✓
T_IRQ → PA3 ✓
T_DO → PA4  ✓
...
```

## 硬件故障排查

### LCD工作但触摸不工作

如果LCD能显示，说明：
- ✓ 模块供电正常
- ✓ SPI通信正常
- ✓ STM32工作正常

问题可能在于：
- ✗ 触摸芯片未供电
- ✗ 触摸信号线接错
- ✗ 触摸芯片损坏（极少见）

### 测试触摸芯片是否工作

运行以下测试代码：
```c
void Test_Touch_ADC(void) {
    LCD_Clear(WHITE);
    char buf[50];
    
    while(1) {
        uint16_t x = TP_Read_AD(CMD_RDX);  // 读X
        uint16_t y = TP_Read_AD(CMD_RDY);  // 读Y
        
        sprintf(buf, "X_ADC: %04d", x);
        LCD_ShowString(10, 100, 16, buf, 0);
        
        sprintf(buf, "Y_ADC: %04d", y);
        LCD_ShowString(10, 120, 16, buf, 0);
        
        HAL_Delay(100);
    }
}
```

**预期结果**:
- 无触摸: X约3900, Y约3900（上拉状态）
- 有触摸: X和Y在200-3900之间变化

**异常情况**:
- 始终是0或4095 → 触摸芯片未工作（检查供电）
- 始终不变 → SPI通信失败（检查PA5/PA7连接）

## 诊断流程图

```
触摸不工作
    ↓
LCD能显示吗？
    ├─ 否 → 检查LCD供电和接线（见Wiring_Guide.md）
    └─ 是 → 触摸VCC有3.3V吗？（万用表测量）
            ├─ 否 → 检查触摸VCC接线
            └─ 是 → T_IRQ会变化吗？（无触摸3.3V，有触摸0V）
                    ├─ 否 → T_IRQ接线错误
                    └─ 是 → 能读到ADC值吗？（运行Test_Touch_ADC）
                            ├─ 否 → PA4/PA5/PA6/PA7接线错误
                            └─ 是 → 坐标不准 → 重新校准（TP_Adjust）
```

## 需要提供的信息

如果以上方法都无法解决，请提供：

1. **硬件信息**
   - 模块照片（正面和背面）
   - 接线照片（能看清杜邦线连接）
   
2. **测试结果**
   - LCD能否显示？
   - 万用表测量触摸VCC电压？
   - T_IRQ在触摸时电压变化？
   - Test_Touch_ADC()的输出值？

3. **代码版本**
   - 使用的是哪个版本的代码？
   - 有做过修改吗？
