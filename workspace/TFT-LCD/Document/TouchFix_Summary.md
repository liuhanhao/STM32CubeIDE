## 问题回顾

### 初始问题
- 触摸功能完全不工作
- PA3-PA7引脚无响应
- 怀疑是接线或代码问题

### 根本原因
经过对比参考程序发现：
1. **底层实现差异** - 软件SPI时序与参考程序不完全一致
2. **初始化时机** - 参考程序在`Touch_Test()`内调用`TP_Init()`
3. **滤波算法** - 参考程序使用5次采样去掉最值后平均

---

## 解决方案

### 1. 完全基于参考程序重写

**touch.c 修改**:
- 软件SPI时序与参考程序完全一致
- 采用参考程序的滤波算法
- 双次读取验证（误差范围50）
- 四点校准算法参照官方例程

**关键代码对比**:

```c
// 参考程序的SPI写时序
void TP_Write_Byte(u8 num) {
    for(count=0; count<8; count++) {  
        if(num&0x80) TDIN=1;  
        else TDIN=0;   
        num<<=1;    
        TCLK=0;      // 先拉低
        TCLK=1;      // 上升沿有效
    }
}

// 参考程序的SPI读时序
u16 TP_Read_AD(u8 CMD) {
    TP_Write_Byte(CMD);
    delay_us(6);  // 转换时间
    
    TCLK=1; TCLK=0;  // 跳过BUSY位
    
    for(count=0; count<16; count++) {
        Num<<=1;
        TCLK=0;  // 下降沿有效
        TCLK=1;
        if(DOUT) Num++;
    }
    Num>>=4;  // 只有高12位有效
    return Num;
}
```

### 2. 简化main.c

参照参考程序的简洁风格：
- 在`Touch_Test()`内部初始化触摸
- 实现简单的画点逻辑
- 支持颜色切换和清屏

## 经验总结

### 关键要点
1. **参考程序的价值** - 官方例程的时序经过验证，不要随意修改
2. **滤波的重要性** - 电阻式触摸屏ADC值抖动大，必须滤波
3. **校准的必要性** - 不同批次触摸屏ADC范围不同，必须校准

### 开发建议
- 先用参考程序验证硬件正常
- 理解原理后再进行代码重构
- 保留参考程序的核心算法（时序、滤波、校准）
