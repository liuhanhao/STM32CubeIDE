# UART 串行通讯协议

## 概述

UART（Universal Asynchronous Receiver/Transmitter，通用异步收发器）是最基础的串行通讯协议。它是**异步**的，意味着通讯双方不共享时钟信号，而是通过约定好的波特率来同步数据。

- 通讯方式：异步、全双工
- 所需引脚：TX（发送）、RX（接收），可选 RTS/CTS 用于硬件流控
- 典型波特率：9600、115200、921600 bps

---

## 帧结构

UART 以"帧"为单位传输数据，每帧结构如下：

```
空闲状态（高电平）
    │
    ▼
┌───┬──────────────────┬──────┬───┐
│ S │  D0 D1 D2 D3 D4 D5 D6 D7 │  P  │ STOP │
└───┴──────────────────┴──────┴───┘
  起始位      数据位（8位）      校验位  停止位
  (低电平)                    (可选)  (高电平)
```

| 字段 | 说明 |
|------|------|
| 起始位 | 1位低电平，通知接收方数据开始 |
| 数据位 | 通常 8 位，LSB（最低位）先发 |
| 校验位 | 可选，奇校验/偶校验/无校验 |
| 停止位 | 1位或2位高电平，表示帧结束 |

## 通讯过程详解

### 1. 初始化阶段

通讯双方必须提前约定相同的参数（称为"串口配置"）：

```
波特率：115200
数据位：8
停止位：1
校验位：无
```

> 如果任何一个参数不匹配，接收方将无法正确解析数据。

### 2. 发送过程（以发送字符 'A' = 0x41 = 0b01000001 为例）

**步骤一：** 总线空闲时保持高电平

```
TX: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
```

**步骤二：** 拉低 TX 发出起始位（持续 1/波特率 秒）

```
TX: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾_
                          ↑起始位
```

**步骤三：** 依次发送 8 位数据（LSB 先发）

'A' = 0x41 = 0b01000001，LSB 先发顺序为：1,0,0,0,0,0,1,0

```
TX: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾_  1  0  0  0  0  0  1  0
                          S  D0 D1 D2 D3 D4 D5 D6 D7
```

**步骤四：** 发送停止位（高电平），帧结束

```
TX: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾_  1  0  0  0  0  0  1  0  ‾
                          S                          STOP
```

### 3. 接收过程

接收方检测到 TX 从高变低（起始位），开始计时：
- 在每个位的中间时刻采样，判断是 0 还是 1
- 采集完 8 位数据后，等待停止位
- 将数据存入接收缓冲区，触发中断通知 CPU

## 完整通讯示例

### 示例1：单片机向 PC 发送字符串

**场景：** 单片机通过 UART 向 PC 发送字符串 "Hi\n"

**硬件连接：**

```
单片机                    USB转串口模块（PC）
  TX ──────────────────→  RX
  RX ←──────────────────  TX
  GND ─────────────────── GND
```

> 注意：TX 接对方 RX，RX 接对方 TX，交叉连接！

**发送 'H'（0x48）、'i'（0x69）、'\n'（0x0A）三个字节的完整时序：**

```
时间轴 →（波特率115200，每位约8.68μs）

         空闲   ←── 'H' ──→   ←── 'i' ──→   ←── '\n' ──→   空闲
TX:  ‾‾‾‾‾‾‾‾_ 0 0 0 1 0 0 1 0 ‾_ 1 0 0 1 0 1 1 0 ‾_ 0 1 0 1 0 0 0 0 ‾‾‾‾‾‾
              S D0          D7 P S D0          D7 P S D0          D7 P
              ↑起始位           ↑停止位
```

字节之间可以紧接着发送（停止位结束后立即发下一个起始位），也可以有间隔。

**代码示例：**

```c
// 初始化 UART
UART_Init(115200, 8, NONE, 1);  // 波特率115200，8数据位，无校验，1停止位

// 发送字符串 "Hi\n"
char *msg = "Hi\n";
for (int i = 0; msg[i] != '\0'; i++) {
    UART_SendByte(msg[i]);  // 逐字节发送，等待上一字节发完再发下一个
}
```

### 示例2：单片机与 PC 双向通讯（AT 指令风格）

**场景：** PC 发送命令，单片机解析后回复结果

```
PC（串口助手）                    单片机
      │                               │
      │── "LED_ON\r\n" ──────────────→│ 接收中断触发，读取数据
      │                               │ 解析命令：LED_ON
      │                               │ 执行：点亮 LED
      │←── "OK\r\n" ─────────────────│ 回复确认
      │                               │
      │── "READ_TEMP\r\n" ───────────→│
      │                               │ 读取温度传感器：25.6°C
      │←── "TEMP=25.6\r\n" ──────────│
```

**接收端代码（中断方式，推荐）：**

```c
// 接收缓冲区
#define RX_BUF_SIZE 64
uint8_t rxBuf[RX_BUF_SIZE];
uint8_t rxIndex = 0;
uint8_t rxComplete = 0;

// UART 接收中断回调（每收到1字节触发一次）
void UART_RxCallback(uint8_t byte) {
    if (byte == '\n') {
        // 收到换行符，一条命令接收完毕
        rxBuf[rxIndex] = '\0';
        rxComplete = 1;
        rxIndex = 0;
    } else if (byte != '\r') {
        // 存入缓冲区（忽略\r）
        if (rxIndex < RX_BUF_SIZE - 1) {
            rxBuf[rxIndex++] = byte;
        }
    }
}

// 主循环处理命令
void Main_Loop(void) {
    while (1) {
        if (rxComplete) {
            rxComplete = 0;
            // 解析命令
            if (strcmp((char*)rxBuf, "LED_ON") == 0) {
                LED_On();
                UART_SendString("OK\r\n");
            } else if (strcmp((char*)rxBuf, "READ_TEMP") == 0) {
                float temp = ReadTemperature();
                char reply[32];
                snprintf(reply, sizeof(reply), "TEMP=%.1f\r\n", temp);
                UART_SendString(reply);
            } else {
                UART_SendString("ERROR\r\n");
            }
        }
    }
}
```

### 示例3：单片机与蓝牙模块通讯（AT 指令配置）

**场景：** 通过 UART 配置 HC-05 蓝牙模块的名称和波特率

```
单片机（UART1）                   HC-05 蓝牙模块
      │                               │
      │── "AT\r\n" ─────────────────→│ 测试连接
      │←── "OK\r\n" ─────────────────│
      │                               │
      │── "AT+NAME=MySensor\r\n" ───→│ 设置蓝牙名称
      │←── "OK\r\n" ─────────────────│
      │                               │
      │── "AT+BAUD=4\r\n" ──────────→│ 设置波特率为9600（4=9600）
      │←── "OK:4\r\n" ───────────────│
```

```c
// 发送 AT 指令并等待响应
bool AT_SendCommand(const char *cmd, const char *expected, uint32_t timeout_ms) {
    UART_SendString(cmd);       // 发送指令
    UART_SendString("\r\n");

    // 等待接收响应（超时机制）
    uint32_t start = GetTick();
    while (!rxComplete) {
        if (GetTick() - start > timeout_ms) {
            return false;       // 超时
        }
    }
    rxComplete = 0;
    return strstr((char*)rxBuf, expected) != NULL;
}

// 配置蓝牙模块
void BT_Configure(void) {
    AT_SendCommand("AT", "OK", 1000);
    AT_SendCommand("AT+NAME=MySensor", "OK", 1000);
    AT_SendCommand("AT+BAUD=4", "OK", 1000);
}
```

## 波特率与误差

波特率决定每个位的持续时间：

```
位时间 = 1 / 波特率
波特率 115200 → 位时间 ≈ 8.68 μs
```

UART 允许最大约 **±2%** 的波特率误差，超过则通讯失败。

## 流量控制（可选）

当接收方处理速度跟不上发送方时，需要流控：

### 软件流控（XON/XOFF）
- 接收方缓冲区满时发送 `0x13`（XOFF）通知对方暂停
- 缓冲区空闲后发送 `0x11`（XON）通知对方继续

### 硬件流控（RTS/CTS）
```
单片机                    对方设备
  RTS ──────────────────→  CTS   （我准备好发送了）
  CTS ←──────────────────  RTS   （我准备好接收了）
```

## 常见问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 收到乱码 | 波特率不匹配 | 检查双方波特率配置 |
| 数据丢失 | 缓冲区溢出 | 启用流控或提高处理速度 |
| 通讯不通 | TX/RX 未交叉 | 检查接线 |
| 帧错误 | 停止位不匹配 | 统一停止位配置 |
