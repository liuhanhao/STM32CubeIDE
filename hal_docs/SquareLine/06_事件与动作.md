# SquareLine Studio — 事件与动作

## 一、事件系统概述

SquareLine Studio 的事件系统对应 LVGL 的 `lv_obj_add_event_cb()`。
设计时绑定好事件，导出后 SquareLine 自动生成回调函数框架，
业务逻辑填入对应的回调函数即可。

---

## 二、添加事件

1. 选中控件
2. 检查器面板切换到 **Events** Tab
3. 点击 **+ Add Event**
4. 选择触发条件（Trigger）和执行动作（Action）

---

## 三、触发条件（Trigger）

| Trigger | 说明 |
|---------|------|
| **Clicked** | 点击（按下后抬起）|
| **Pressed** | 按下 |
| **Released** | 抬起 |
| **Long Pressed** | 长按 |
| **Long Pressed Repeat** | 长按持续触发 |
| **Value Changed** | 值改变（Slider、Switch、Roller 等）|
| **Focused** | 获得焦点 |
| **Defocused** | 失去焦点 |
| **Screen Loaded** | 屏幕加载完成（加在屏幕根节点上）|
| **Screen Unloaded** | 屏幕卸载 |

---

## 四、内置动作（Action）

### 4.1 屏幕跳转

| 动作 | 说明 |
|------|------|
| **Load Screen** | 跳转到指定屏幕 |

设置项：
- **Screen**：目标屏幕名称
- **Animation**：跳转动画
- **Speed**：动画时长（ms）
- **Delay**：延迟执行（ms）

**可用动画：**

| 动画 | 说明 |
|------|------|
| None | 无动画，立即切换 |
| Fade On | 淡入 |
| Over Left | 从右向左滑入 |
| Over Right | 从左向右滑入 |
| Over Top | 从下向上滑入 |
| Over Bottom | 从上向下滑入 |
| Move Left | 向左平移切换 |
| Move Right | 向右平移切换 |

### 4.2 控件操作

| 动作 | 说明 |
|------|------|
| **Change Screen** | 切换屏幕（同 Load Screen）|
| **Increment Arc** | 弧形控件增加值 |
| **Decrement Arc** | 弧形控件减少值 |
| **Modify Flag** | 修改控件标志（Hidden/Disabled 等）|
| **Set Opacity** | 设置控件透明度 |
| **Play Animation** | 播放指定动画 |
| **Pause Animation** | 暂停动画 |

### 4.3 调用自定义函数

选择动作 **Call Function**，输入函数名：

```
Function Name:  my_button_click_handler
```

导出后会生成：

```c
/* ui_events.c */
void my_button_click_handler(lv_event_t *e)
{
    // 在这里填入业务逻辑
}
```

## 五、一个事件对应多个动作

一个触发条件可以绑定多个动作，按顺序执行：

```
Button_Start → Clicked:
  1. Load Screen → Screen_Home（动画：Over Left，300ms）
  2. Call Function → start_sensor_task
```

## 六、导出的事件代码

SquareLine Studio 导出两类文件：

### ui.c / ui_Screen_xxx.c（自动生成，不要手动修改）

```c
/* 自动生成的事件绑定代码 */
lv_obj_add_event_cb(ui_Button_Start, ui_event_Button_Start,
                     LV_EVENT_ALL, NULL);
```

```c
/* 自动生成的事件分发函数 */
void ui_event_Button_Start(lv_event_t *e)
{
    lv_event_code_t event_code = lv_event_get_code(e);
    lv_obj_t *target = lv_event_get_target(e);

    if (event_code == LV_EVENT_CLICKED) {
        _ui_screen_change(&ui_Screen_Home,
                          LV_SCR_LOAD_ANIM_OVER_LEFT, 300, 0, &ui_Screen_Home_screen_init);
        my_button_click_handler(e);  // ← 调用你的函数
    }
}
```

### ui_events.c（手动填入业务逻辑）

```c
/* ui_events.c — 这个文件是你写逻辑的地方 */
#include "ui.h"

void my_button_click_handler(lv_event_t *e)
{
    lv_event_code_t code = lv_event_get_code(e);
    if (code == LV_EVENT_CLICKED) {
        // 你的业务逻辑
        start_motor();
        update_display();
    }
}
```

> ⚠️ **规则：**
> - `ui_Screen_xxx.c` 是 SquareLine 自动生成的，每次导出都会覆盖，**不要手动修改**
> - `ui_events.c` 是你的业务逻辑文件，**只写这个文件**，导出不会覆盖

## 七、跨屏幕访问控件

如果需要在事件回调里访问其他屏幕的控件，控件变量需要声明为 `extern`：

```c
/* ui.h 中已自动导出 */
extern lv_obj_t *ui_Label_Status;   // 在 Screen_Home 上的标签

/* 在 ui_events.c 中直接使用 */
void update_status_handler(lv_event_t *e)
{
    lv_label_set_text(ui_Label_Status, "运行中");
}
```

## 八、常见事件组合示例

### 按钮跳转页面

```
Button_Setting → Clicked → Load Screen → Screen_Setting（Fade On，300ms）
Button_Back    → Clicked → Load Screen → Screen_Home（Over Right，300ms）
```

### Switch 控制显示/隐藏

```
Switch_Light → Value Changed → Call Function → toggle_light_handler
```

```c
void toggle_light_handler(lv_event_t *e)
{
    lv_obj_t *sw = lv_event_get_target(e);
    if (lv_obj_has_state(sw, LV_STATE_CHECKED)) {
        lv_obj_clear_flag(ui_Panel_Light, LV_OBJ_FLAG_HIDDEN);
        gpio_set_level(LIGHT_PIN, 1);
    } else {
        lv_obj_add_flag(ui_Panel_Light, LV_OBJ_FLAG_HIDDEN);
        gpio_set_level(LIGHT_PIN, 0);
    }
}
```

### Slider 更新 Label

```
Slider_Brightness → Value Changed → Call Function → slider_change_handler
```

```c
void slider_change_handler(lv_event_t *e)
{
    lv_obj_t *slider = lv_event_get_target(e);
    int32_t value = lv_slider_get_value(slider);
    lv_label_set_text_fmt(ui_Label_Value, "%d%%", value);
}
```
