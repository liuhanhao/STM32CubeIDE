## 当前模式概述

### 修改内容
- ✅ 每次启动都执行四点校准 `TP_Adjust()`
- ✅ 校准完成后直接进入画板，不保存到Flash
- ✅ CAL按钮可重新校准（运行时有效，断电丢失）
- ✅ Flash读写代码已注释，保护学习用芯片

### 上电流程
```
上电 → LCD初始化 → 触摸初始化 → 四点校准 → 进入画板
```

### 校准步骤
1. 屏幕显示红色十字准心（左上角）
2. 用触摸笔/手指**精确点击**准心中心
3. 准心移动到右上角，继续点击
4. 准心移动到左下角，继续点击  
5. 准心移动到右下角，继续点击
6. 显示 "Touch Screen Adjust OK!"
7. 进入画板界面

### 校准注意事项
- ⚠️ **必须精确点击十字中心**，偏差会影响后续使用精度
- ⚠️ 如果某次点击失败，会自动重新开始（回到第一个点）
- ⚠️ 如果10秒内没有操作，自动退出校准
- ✅ 校准完成后断电会丢失，下次需要重新校准

### CAL按钮行为
点击画板底部的"CAL"按钮：
- 重新执行四点校准
- 校准完成后刷新画板界面
- 新的校准参数立即生效（但断电丢失）

## 如何启用生产级Flash校准

当你的项目完成开发，准备量产时：

### 方法：修改代码启用Flash功能

在 `main.c` 的 Touch_Test() 函数中找到：
```c
/* ========== Flash保存代码（已注释，学习阶段避免频繁擦写） ========== */
#if 0  /* 如需启用Flash保存，将0改为1 */
```

**改为：**
```c
#if 1  /* 如需启用Flash保存，将0改为1 */
```

然后在前面添加Flash读取逻辑（参考之前的完整版本）。

### 生产阶段的完整流程
1. **出厂前**：烧录程序 → 执行一次校准 → 保存到Flash → 封装出货
2. **用户使用**：上电 → 从Flash加载校准数据 → 直接进入画板
3. **维修场景**：点击CAL按钮 → 重新校准 → 保存到Flash

## 为什么需要校准？

### 不校准的问题
之前尝试的"不校准模式"使用简单线性映射：
```c
screen_x = adc_x * (240 / 4096)  // 太简化了
```

**实际问题：**
- ❌ 触摸屏ADC值不是从0开始（通常200-3900左右）
- ❌ 不同批次的触摸屏ADC范围不同
- ❌ 边缘区域非线性失真严重
- ❌ X/Y可能需要互换或镜像翻转

### 四点校准的优势
通过测量四个角点的ADC值：
```
点1(20,20)     → ADC(300, 250)
点2(220,20)    → ADC(3800, 280)  
点3(20,300)    → ADC(280, 3850)
点4(220,300)   → ADC(3750, 3900)
```

计算出精确的转换参数：
```c
xfac = 200 / (3800 - 300) = 0.0571  // X缩放系数
xoff = 20 - 300 * 0.0571 = 2.87     // X偏移量
// Y方向同理
```

**效果：**
- ✅ 自动适应不同批次的触摸屏
- ✅ 自动处理ADC偏移
- ✅ 精度高达±1像素
- ✅ 自动检测并修正X/Y翻转

## 常见问题

### Q1: 每次上电校准太麻烦，怎么办？
A: 学习阶段这是正常的。等学完后启用Flash功能，就只需要校准一次。

### Q2: 校准时点击不准确怎么办？
A: 会自动重来。系统会验证四个点的距离关系，如果不合格会重新开始。

### Q3: 能不能跳过校准直接用？
A: 不能。前面已经试过不校准模式，效果很差（你说"乱七八糟的"）。

### Q4: 学习阶段每天要烧录很多次，会伤芯片吗？
A: 会的。但这是开发必要的代价。建议：
   - 多用仿真器调试，少烧录
   - 代码稳定后再频繁测试
   - 有条件的话用几块芯片轮流测试

### Q5: 生产阶段Flash会被频繁擦写吗？
A: 不会。正常使用下，用户几乎不会点CAL按钮。即使每年重新校准1次，10000次寿命也够用10000年。

## 校准原理详解

### 四点法数学模型
假设触摸屏坐标系为 (tx, ty)，LCD屏幕坐标系为 (sx, sy)，转换关系：
```
sx = tx * xfac + xoff
sy = ty * yfac + yoff
```

通过四个已知点求解 xfac, xoff, yfac, yoff：
```
点1: (sx1, sy1) = (20, 20)     → (tx1, ty1)
点2: (sx2, sy2) = (220, 20)    → (tx2, ty2)
点3: (sx3, sy3) = (20, 300)    → (tx3, ty3)  
点4: (sx4, sy4) = (220, 300)   → (tx4, ty4)
```

求解：
```c
xfac = (sx2 - sx1) / (tx2 - tx1)
xoff = (sx1 + sx2) / 2 - xfac * (tx1 + tx2) / 2

yfac = (sy3 - sy1) / (ty3 - ty1)  
yoff = (sy1 + sy3) / 2 - yfac * (ty1 + ty3) / 2
```

这就是 `TP_Adjust()` 函数做的事情！
