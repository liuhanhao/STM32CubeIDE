# LIN 局域互联网络协议

## 概述

LIN（Local Interconnect Network，局域互联网络）是一种**单线、低成本**的串行通讯协议，专为汽车辅助系统设计。它是 CAN 总线的低成本补充，用于对速度要求不高、成本敏感的场合。

- 通讯方式：异步、半双工、单主多从
- 所需引脚：**1根**数据线（LIN 线）+ GND
- 典型速度：1 kbps ～ 20 kbps（标准最高 20 kbps）
- 最大节点数：1个主机 + 最多 **16个**从机
- 电压：12V 总线电平（汽车系统）

---

## 与 CAN 的对比定位

```
CAN 总线：高速、高可靠、贵
  → 用于发动机、ABS、安全气囊等关键系统

LIN 总线：低速、低成本、简单
  → 用于车窗、座椅、后视镜、雨刮器、车内照明等辅助系统
```

## 帧结构

LIN 帧由**帧头（Header）**和**响应（Response）**两部分组成：

```
┌──────────────────────────────┬──────────────────────────────────────┐
│         帧头（主机发送）       │         响应（主机或从机发送）         │
├──────┬──────────┬────────────┼──────────────────────┬──────────────┤
│ 间隔 │  同步字节  │  受保护ID  │    数据（1～8字节）   │   校验和     │
│Break │  0x55    │  1字节     │                      │   1字节      │
└──────┴──────────┴────────────┴──────────────────────┴──────────────┘
```

### 各字段详解

**间隔字段（Break Field）**
- 至少 13 个显性位（低电平），后跟 1 个隐性位
- 作用：通知所有从机"一个新帧开始了"

**同步字节（Sync Byte）**
- 固定值 `0x55`（0b01010101）
- 作用：从机通过测量此字节的位时间来同步波特率

**受保护ID（Protected Identifier）**
- 6位帧ID（0x00～0x3F）+ 2位奇偶校验
- 帧ID决定：谁来响应、数据长度、数据方向

**数据字段**
- 1～8字节，长度由帧ID隐含决定

**校验和**
- 经典校验：数据字节之和取反（LIN 1.x）
- 增强校验：ID + 数据字节之和取反（LIN 2.x，推荐）

## 通讯过程详解

LIN 是严格的**主从架构**：
- 主机负责发送帧头，控制总线节奏
- 从机根据帧ID决定是否响应，以及如何响应

### 主机发布命令（主机发送数据给从机）

```
主机                              从机（车窗电机，ID=0x10）
  │                                    │
  │── Break（13位低电平）──────────────→│ 所有从机检测到帧开始
  │── Sync（0x55）────────────────────→│ 从机同步波特率
  │── Protected ID（0x10）────────────→│ 从机识别ID=0x10，这是给我的命令
  │── 数据[0x01, 0x64]────────────────→│ 命令：方向=上升(0x01)，速度=100%(0x64)
  │── 校验和────────────────────────→│ 从机验证数据正确性
  │                                    │ 从机执行：驱动电机上升
```

### 主机请求数据（从机发送数据给主机）

```
主机                              从机（温度传感器，ID=0x20）
  │                                    │
  │── Break ──────────────────────────→│
  │── Sync（0x55）────────────────────→│
  │── Protected ID（0x20）────────────→│ 从机识别ID=0x20，主机要读我的数据
  │                                    │── 数据[0x01, 0x1A]──────────────→│ 从机发送温度：26°C
  │                                    │── 校验和──────────────────────→│
  │←─ 接收数据 ────────────────────────│
```

## 完整通讯示例

**场景：** 汽车 BCM（车身控制模块）通过 LIN 控制车窗和读取车内温度

### 帧ID分配表

| 帧ID | 方向 | 数据含义 |
|------|------|----------|
| 0x10 | 主→从 | 车窗控制命令 |
| 0x11 | 主→从 | 后视镜控制命令 |
| 0x20 | 从→主 | 车内温度数据 |
| 0x21 | 从→主 | 车窗位置反馈 |

### 控制车窗上升

```
发送帧：
  Break: 低电平 ×13位
  Sync:  0x55
  PID:   0x50（ID=0x10，奇偶校验后=0x50）
  Data:  0x01（方向：上升）0x64（速度：100%）
  Check: ~(0x50 + 0x01 + 0x64) & 0xFF = 0xEA（增强校验）

完整字节流：[Break][0x55][0x50][0x01][0x64][0xEA]
```

### 读取车内温度

```
主机发送帧头：
  [Break][0x55][0x60]（ID=0x20，PID=0x60）

从机（温度传感器）响应：
  Data:  0x1A（26°C）0x00（小数部分）
  Check: ~(0x60 + 0x1A + 0x00) & 0xFF = 0x85

完整响应：[0x1A][0x00][0x85]
```

### 代码示例

```c
// LIN 主机发送帧头
void LIN_SendHeader(uint8_t frameID) {
    // 发送 Break 字段（使用低波特率发送0x00实现）
    UART_SetBaudrate(BAUD_BREAK);    // 降低波特率
    UART_SendByte(0x00);             // 发送0x00产生Break
    UART_SetBaudrate(BAUD_NORMAL);   // 恢复正常波特率

    // 发送同步字节
    UART_SendByte(0x55);

    // 发送受保护ID（加入奇偶校验位）
    uint8_t pid = LIN_CalcPID(frameID);
    UART_SendByte(pid);
}

// LIN 主机发送数据（主机发布命令）
void LIN_SendData(uint8_t frameID, uint8_t *data, uint8_t len) {
    LIN_SendHeader(frameID);

    for (int i = 0; i < len; i++) {
        UART_SendByte(data[i]);
    }

    // 发送增强校验和
    uint8_t checksum = LIN_CalcChecksum(LIN_CalcPID(frameID), data, len);
    UART_SendByte(checksum);
}

// 控制车窗示例
void Window_Control(uint8_t direction, uint8_t speed) {
    uint8_t cmd[2] = {direction, speed};
    LIN_SendData(0x10, cmd, 2);  // 发送到ID=0x10（车窗控制）
}
```

## 调度表（Schedule Table）

LIN 主机按照预定的**调度表**周期性发送帧头，确保总线有序：

```
调度表示例（周期 10ms）：
时间槽 0ms:  发送 ID=0x10（车窗命令）
时间槽 2ms:  发送 ID=0x11（后视镜命令）
时间槽 4ms:  发送 ID=0x20（读温度）
时间槽 6ms:  发送 ID=0x21（读车窗位置）
时间槽 8ms:  空闲
时间槽 10ms: 循环重新开始
```

## 典型应用

- 车窗升降控制
- 电动后视镜调节

- 空调风量/温度控制面板
- 方向盘按键模块
