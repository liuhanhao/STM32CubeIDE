## 🎯 项目概述

### 硬件组成

| 组件 | 型号 | 功能 |
|------|------|------|
| **开发板** | STM32F103C8T6 | 主控芯片（72MHz Cortex-M3）|
| **LCD屏幕** | ILI9341 | 2.4寸 240×320 TFT显示屏 |
| **触摸控制器** | XPT2046/TSC2046 | 4线电阻式触摸，12位ADC |
| **稳压器** | ME6206A33 | 3.3V LDO（300mA）|

### 功能特性

- ✅ 240×320 RGB565 全彩显示
- ✅ 4线电阻式触摸（12位精度）
- ✅ 四点触摸校准算法
- ✅ 实时触摸绘图（2ms采样间隔）
- ✅ 多颜色画笔切换
- ✅ 绘图区域清除
- ✅ 运行时重新校准

### 接口分配

**LCD显示（SPI2 + GPIO）**
- PB13 - SCK（时钟）
- PB15 - MOSI（数据输入）
- PB14 - MISO（数据输出，可选）
- PB11 - CS（片选）
- PB12 - RST（复位）
- PB10 - DC（数据/命令）
- PB9  - BL（背光）

**触摸屏（软件SPI）**
- PA7 - T_CLK（时钟）
- PA5 - T_DIN（数据输入）
- PA4 - T_DO（数据输出）
- PA6 - T_CS（片选）
- PA3 - T_IRQ（中断）

## 🚀 快速上手

### 1. 硬件连接

参考 [接线指南](./Wiring_Guide.md) 完成所有接线，重点检查：
- ✓ VCC/GND供电
- ✓ LCD信号线（PB组）
- ✓ 触摸信号线（PA组）

### 2. 编译烧录

```bash
# 在STM32CubeIDE中
1. 打开项目
2. Build Project (Ctrl+B)
3. Run/Debug (F11)
```

### 3. 首次使用

上电后会自动进入**四点校准模式**：
1. 屏幕显示红色十字准心
2. 用触摸笔/手指**精确点击**准心中心
3. 依次点击4个角的准心（左上→右上→左下→右下）
4. 显示 "Touch Screen Adjust OK!"
5. 进入画板界面

### 4. 功能测试

- **绘图**: 触摸屏幕任意位置绘制轨迹
- **换色**: 点击底部颜色按钮（7种颜色）
- **清屏**: 点击底部 "CLR" 按钮
- **重新校准**: 点击底部 "CAL" 按钮

## 📊 系统架构

### 渲染机制

```
┌─────────────────────────────────────────┐
│              渲染流程                   │
├─────────────────────────────────────────┤
│                                         │
│  STM32 主循环（2ms周期）                │
│      ↓                                  │
│  1. 扫描触摸 TP_Scan()                  │
│      ├─ 读取ADC值（X/Y坐标）            │
│      └─ 校准转换（物理→屏幕坐标）        │
│      ↓                                  │
│  2. 判断触摸状态                         │
│      ├─ 有触摸：绘制点/线                │
│      └─ 无触摸：跳过                     │
│      ↓                                  │
│  3. 实时绘制（逐点/逐线）                │
│      └─ LCD_DrawLine()                  │
│          └─ 通过SPI发送像素数据          │
│              └─ ILI9341 GRAM 更新       │
│                  └─ 屏幕立即显示         │
│                                         │
│  ⚠️ 无帧缓冲，STM32直接渲染到LCD        │
│  ⚠️ 增量更新，只绘制变化部分             │
│  ⚠️ 全屏刷新约270ms（320×240像素）      │
│                                         │
└─────────────────────────────────────────┘
```

**关键特性**:
- **无帧缓冲**: LCD没有独立显存，STM32逐点发送
- **直接渲染**: 每个像素通过SPI实时发送
- **增量更新**: 只绘制触摸轨迹，不刷新整屏
- **性能限制**: SPI 4.5MHz，理论最大帧率~3.7fps

### 触摸校准原理

```
四点校准法：
┌─────────────────────────┐
│ P1(20,20)    P2(220,20) │  物理坐标→屏幕坐标
│   ✚            ✚        │  
│                          │  sx = tx × xfac + xoff
│                          │  sy = ty × yfac + yoff
│                          │
│ P3(20,300)  P4(220,300) │  通过4个已知点求解：
│   ✚            ✚        │  xfac, xoff, yfac, yoff
└─────────────────────────┘
```

## 🔧 常见问题

### LCD不显示
1. 检查VCC/GND供电
2. 检查PB组引脚连接
3. 确认RST引脚为高电平

### 触摸无响应
1. **最常见**: 触摸VCC未接 → 检查供电
2. 检查PA组引脚连接
3. 用万用表测T_IRQ电压（无触摸3.3V，有触摸0V）

### 触摸坐标不准
1. 重新执行四点校准
2. 点击准心时要精确对准中心
3. 检查T_DO(PA4)连接质量

更多问题参考 [触摸故障排查](./Touch_Troubleshooting.md)
