## 1. 供电接线

### 1.1 必须的供电连接

```
LCD模块      →    STM32开发板
─────────────────────────────
VCC(3.3V)   →    3.3V引脚
GND         →    GND引脚
```

### 1.2 重要提示

- ✅ **必须接3.3V**，不能接5V
- ✅ **VCC和GND必须接**，否则模块不工作
- ✅ 如果LCD和触摸是一体模块，VCC/GND可以共用
- ❌ **绝对不能接5V**，会损坏芯片！

### 1.3 在STM32F103C8T6蓝色板上的位置

```
     ┌─────────────────┐
     │  STM32F103C8T6  │
     │                 │
     │  ⚡ 3.3V  ──→  LCD VCC
     │  ⚡ GND   ──→  LCD GND
     │                 │
     └─────────────────┘
```

## 2. LCD显示接线

### 2.1 SPI接口连接

| LCD信号 | 功能说明 | STM32引脚 | 引脚类型 |
|---------|---------|-----------|---------|
| **CS** | 片选（低电平有效） | **PB11** | 输出 |
| **RST** | 复位（低电平复位） | **PB12** | 输出 |
| **DC/RS** | 数据/命令选择 | **PB10** | 输出 |
| **SDI(MOSI)** | SPI数据输入 | **PB15** | 输出 |
| **SCK** | SPI时钟 | **PB13** | 输出 |
| **LED(BL)** | 背光控制 | **PB9** | 输出 |
| **SDO(MISO)** | SPI数据输出（可选） | **PB14** | 输入 |

### 2.2 LCD控制器：ILI9341

- 分辨率：240×320像素
- 颜色：RGB565（65K色）
- 接口：4线SPI模式

### 2.3 背光控制

**选项A：GPIO控制（推荐）**
```c
// PB9控制背光
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_9, GPIO_PIN_SET);   // 点亮
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_9, GPIO_PIN_RESET); // 熄灭
```

**选项B：常亮**
```
LED(BL) 直接接 3.3V → 背光常亮
```

## 3. 触摸屏接线

### 3.1 完整接线表

| 触摸信号 | 功能说明 | STM32引脚 | 引脚类型 |
|---------|---------|-----------|---------|
| **VCC** | 触摸IC供电（3.3V） | **3.3V** | 电源 |
| **GND** | 触摸IC地 | **GND** | 地 |
| **T_IRQ** | 触摸中断（低有效） | **PA3** | 输入 |
| **T_DO** | 软件SPI输出(MISO) | **PA4** | 输入 |
| **T_DIN** | 软件SPI输入(MOSI) | **PA5** | 输出 |
| **T_CS** | 片选（低有效） | **PA6** | 输出 |
| **T_CLK** | 软件SPI时钟 | **PA7** | 输出 |

### 3.2 触摸控制器：XPT2046/TSC2046

- 12位ADC精度（0-4095）
- 4线电阻式触摸
- 软件SPI接口
- 工作电压：2.2V ~ 5.25V（本项目使用3.3V）

### 3.3 触摸工作原理

```
无触摸时：T_IRQ = 高电平(3.3V)
有触摸时：T_IRQ = 低电平(0V)

MCU检测到T_IRQ低电平 → 通过SPI读取X/Y坐标
```

## 5. 常见问题

### Q1: LCD不显示或显示异常

**可能原因：**
- VCC/GND未接或接错
- SPI引脚接错
- 复位未释放（RST一直为低）

**排查方法：**
1. 用万用表测量VCC是否有3.3V
2. 检查所有信号线是否接对
3. 确保RST引脚初始化为高电平

### Q2: 触摸完全无反应

**可能原因：**
- 触摸VCC/GND未接（最常见！）
- T_IRQ引脚接错
- 触摸IC损坏

**排查方法：**
1. **首先检查触摸VCC是否有供电！**
2. 运行引脚扫描程序，触摸时观察哪个引脚变化
3. 用万用表测量T_IRQ引脚电压（无触摸3.3V，有触摸0V）

### Q3: 触摸有反应但坐标不准

**可能原因：**
- 未进行触摸校准
- X/Y轴方向反了
- SPI通信不稳定

**解决方法：**
1. 调用 `TP_Adjust()` 进行四点校准
2. 如果方向反了，调整 `CMD_RDX` 和 `CMD_RDY`
3. 检查T_DO(PA4)连接是否良好

### Q4: 背光不亮

**可能原因：**
- LED引脚未接或接错
- PB9初始化为低电平

**解决方法：**
```c
// 确保PB9初始化为输出并置高
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_9, GPIO_PIN_SET);
```

或者直接把LED接3.3V常亮

### Q5: 如何确认接线正确？

**方法1：引脚扫描程序**
```
运行本项目的 Touch_Test() 函数
触摸屏幕，观察哪个PA引脚从GREEN变RED
该引脚就是T_IRQ
```

**方法2：万用表测量**
```
黑表笔接GND
红表笔依次测量触摸排针

VCC针：3.3V
GND针：0V
T_IRQ针：无触摸3.3V，触摸时0V
其他针：不确定
```

## 5. 测试步骤

### 步骤1：供电测试 ⚡
- 接好VCC和GND
- 用万用表测量LCD模块VCC引脚是否有3.3V

### 步骤2：LCD显示测试 📺
- 接好所有LCD信号线（PB9-15）
- 烧录程序
- LCD应能显示画面和背光点亮

### 步骤3：触摸IRQ测试 👆
- 接好触摸VCC/GND（如果是独立排针）
- 接好触摸信号线（PA3-7）
- 触摸屏幕，观察调试信息中PA3是否从高变低

### 步骤4：触摸坐标测试 🎯
- 运行完整触摸程序
- 触摸屏幕应能画出轨迹
- 观察坐标数值是否合理（X: 0-239, Y: 0-319）

### 步骤5：触摸校准 ✨
- 每次上电会自动进入四点校准
- 依次精确点击四个红色十字准心
- 校准完成后进入画板界面
