## 基础模块

### HAL Core — HAL 核心
**文档**: [HAL_Core.md](HAL_Core.md)

HAL 库的基础设施层，提供系统节拍、延时、芯片信息等全局功能。所有项目必须首先调用 `HAL_Init()` 和 `SystemClock_Config()`。

**核心特点**
- `HAL_Delay` 是阻塞延时，占用 CPU；多任务并发场景应改用 `HAL_GetTick()` 实现非阻塞计时
- `HAL_GetTick()` 返回毫秒计数，适合超时判断和周期任务调度
- 芯片唯一 ID（`HAL_GetUIDw0/1/2`）可用于序列号生成或设备授权
- `HAL_SuspendTick` / `HAL_ResumeTick` 配合低功耗模式使用，进入睡眠前暂停 SysTick 以降低功耗

**适用场景**
- 任何项目的基础初始化
- 替代 `HAL_Delay` 的非阻塞多任务调度
- 低功耗模式下的 Tick 管理

### HAL RCC — 时钟控制
**文档**: [HAL_RCC.md](HAL_RCC.md)

管理芯片所有时钟源和外设时钟使能，是系统运行的基础。

**核心特点**
- STM32F1 有四个时钟源：HSE（外部晶振）、HSI（内部 RC）、LSE（32.768kHz 晶振）、LSI（内部低速 RC）
- 通常用 HSE（8MHz）经 PLL 9 倍频得到 72MHz 最高主频
- 没有外部晶振时可用 HSI/2 × 16 = 64MHz
- 每个外设使用前**必须先使能其时钟**，忘记是新手最常见的 bug
- APB1 最高 36MHz，APB2 最高 72MHz；ADC 时钟最高 14MHz，需额外配置预分频

**适用场景**

| 场景 | 时钟源选择 |
|------|-----------|
| 正式产品，需要 USB / CAN | 必须用 HSE（USB 需 48MHz 整数倍） |
| 原型开发，节省晶振成本 | HSI |
| RTC 实时时钟 | LSE（精度高）或 LSI（无需晶振） |
| IWDG 独立看门狗 | 固定使用 LSI，无法更改 |

### HAL CORTEX — 内核中断控制
**文档**: [HAL_CORTEX.md](HAL_CORTEX.md)

管理 Cortex-M3 内核的 NVIC 中断优先级系统和 SysTick。

**核心特点**
- 优先级分为抢占优先级和子优先级，推荐使用 `NVIC_PRIORITYGROUP_4`（4 位抢占，0 位子优先级）
- 数值越小优先级越高（0 最高，15 最低）
- 高抢占优先级的中断可以打断正在执行的低优先级中断（嵌套）
- 同抢占优先级的中断不能互相打断，按子优先级顺序响应

**适用场景**
- 配置多个中断的优先级层级（如定时器 > 通信 > 按键）
- 临界区保护（禁用/使能特定中断）
- FreeRTOS 项目需按文档要求设置内核中断优先级下限

## IO 与中断

### HAL GPIO — 通用 IO
**文档**: [HAL_GPIO.md](HAL_GPIO.md)

控制数字输入输出引脚，是所有数字外设操作的基础。

**核心特点**

| 模式 | 特点 | 典型用途 |
|------|------|---------|
| 浮空输入 | 电平由外部决定，无内部上下拉 | 外部有上拉/下拉电阻的信号线 |
| 推挽输出 | 能主动输出高低电平，驱动能力强 | LED、继电器控制、SPI 信号线 |
| 开漏输出 | 只能拉低，释放时高阻态；需外部上拉 | I²C 总线、多设备共享信号线、3.3V→5V 电平转换 |
| 模拟模式 | 完全断开数字逻辑 | ADC 输入引脚、DAC 输出引脚、省功耗的未用引脚 |
| 复用推挽/开漏 | 引脚交给外设（UART/SPI/I2C）控制 | 外设专用引脚 |
| 外部中断 | 边沿触发中断 | 按键、传感器信号检测 |

**适用场景**
- LED、蜂鸣器、继电器等数字量控制 → 推挽输出
- 按键、限位开关等数字量检测 → 输入 + 上拉
- I²C 引脚 → 开漏输出，外接 4.7kΩ 上拉
- ADC/DAC 引脚 → 模拟模式（必须，否则影响精度）

---

### HAL EXTI — 外部中断
**文档**: [HAL_EXTI.md](HAL_EXTI.md)

为 GPIO 引脚提供边沿触发中断能力。实际使用中通常通过 GPIO 配置而非直接操作 EXTI。

**核心特点**
- 每条 EXTI 线对应同一编号的引脚（EXTI0 对应所有端口的 Pin0）
- 同编号引脚（如 PA0 和 PB0）只能选其中一个接入 EXTI0
- PIN0~PIN4 各有独立中断向量；PIN5~PIN9 共用一个；PIN10~PIN15 共用一个
- 支持上升沿、下降沿、双边沿三种触发方式

**适用场景**
- 按键检测（替代轮询，节省 CPU）
- 外部传感器信号变化检测（红外、霍尔传感器）
- 旋转编码器方向判断
- 任何需要对边沿变化立即响应的场景

## 通信外设

### HAL UART — 异步串口
**文档**: [HAL_UART.md](HAL_UART.md)

最常用的调试和通信接口，全双工异步通信。

**核心特点**
- 支持阻塞、中断、DMA 三种传输模式
- 阻塞模式简单但占用 CPU；中断模式适合中等数据量；DMA 模式适合大数据量
- 不定长数据接收最佳方案：中断逐字节接收，或配合"空闲中断"检测帧尾
- `fputc` 重定向后可直接使用 `printf` 打印调试信息

**何时选哪种模式**

| 场景 | 模式 |
|------|------|
| 调试打印、偶发少量输出 | 阻塞发送 |
| 交互式命令行，不知道什么时候来数据 | 中断接收（逐字节） |
| GPS、蓝牙模块的连续数据流 | DMA 循环接收 + 空闲中断 |
| 批量发送日志或固件数据 | DMA 发送 |

**UART vs USART**

STM32 上的 USART 是 UART 的超集，日常当普通异步串口使用时两者用法完全一致。

| 特性 | UART | USART |
|------|------|-------|
| 通信方式 | 仅异步 | 异步 + 同步 |
| 引脚 | RX、TX | RX、TX（同步模式额外加 SCLK） |
| 同步时钟 | ❌ | ✅ SCLK 统一节拍，速度更快、数据更稳 |
| 日常使用 | 接线用法一致，无需区分 | 接线用法一致，无需区分 |

### HAL I2C — I²C 总线
**文档**: [HAL_I2C.md](HAL_I2C.md)

双线串行总线，适合低速、多设备连接场景。

**核心特点**
- 只需两根线（SCL/SDA），多个设备通过 7 位地址区分，挂在同一总线
- 最高速度：标准模式 100kHz，快速模式 400kHz
- 引脚必须配为开漏输出，外接 4.7kΩ 上拉电阻（内部上拉驱动能力不足）
- `Mem_Write` / `Mem_Read` 适合有内部寄存器的设备（如传感器、EEPROM）
- `Master_Transmit` / `Receive` 适合自定义协议或无寄存器设备

**适用场景**
- 各类传感器（MPU6050、BMP280、SHT30 等）
- EEPROM（AT24C 系列）存储少量配置
- OLED 显示屏（SSD1306）
- IO 扩展芯片（PCF8574）
- 多模块系统（只用两根线连接所有设备）

**软件 I2C vs 硬件 I2C**

| 对比项 | 软件 I2C | 硬件 I2C |
|--------|----------|----------|
| 实现方式 | GPIO 手动模拟时序 | 芯片内置 I2C 外设模块 |
| 引脚限制 | ✅ 任意 GPIO | ❌ 固定引脚（如 PB6/PB7） |
| 移植性 | ✅ 换芯片只改 GPIO 操作 | ❌ 依赖 HAL 库，换平台需重写 |
| CPU 占用 | ❌ 高，时序全靠 CPU 等待 | ✅ 低，硬件自动处理，可配合 DMA/中断 |
| 速度 | ❌ 受限于 CPU 执行速度 | ✅ 硬件级，支持标准/快速/超快速模式 |
| 稳定性 | ❌ 中断打断可能导致时序抖动 | ✅ 硬件保证时序精准 |
| 调试难度 | ✅ 代码透明，出错容易定位 | ❌ HAL 封装深，出错不好排查 |

**选用建议**：引脚不够用或需接多个 I2C 设备在不同引脚 → 软件 I2C；追求性能、稳定性 → 硬件 I2C；学习协议原理 → 先写一遍软件 I2C。

**I²C vs SPI 选择**：I²C 引脚少（2 vs 4）但速度慢；SPI 速度快（最高 18Mbps）但每增加一个设备需要一根 CS 线。

### HAL SPI — SPI 总线
**文档**: [HAL_SPI.md](HAL_SPI.md)

高速同步串行总线，适合大数据量传输。

**核心特点**
- 全双工，4 线通信（SCK/MOSI/MISO/CS），CS 由软件手动控制
- STM32F1 SPI1（APB2）最高 36MHz，SPI2/3（APB1）最高 18MHz
- 时钟模式（CPOL/CPHA）需与从设备匹配，绝大多数设备使用 Mode 0
- 支持阻塞、中断、DMA 三种传输模式；大数据量刷屏优先用 DMA

**适用场景**
- SPI Flash 存储（W25Q 系列）：大容量数据存储
- TFT/OLED 显示屏：高速刷屏
- SPI ADC（MCP3208）：高速模拟采集
- 两块 MCU 之间高速互联

**软件 SPI vs 硬件 SPI**

| 对比项 | 软件 SPI | 硬件 SPI |
|--------|----------|----------|
| 实现方式 | GPIO 手动翻转时钟/数据线 | 芯片内置 SPI 外设模块 |
| 引脚限制 | ✅ 任意 GPIO | ❌ 固定引脚（如 PA5/PA6/PA7） |
| 移植性 | ✅ 换芯片只改 GPIO 操作 | ❌ 依赖 HAL 库，换平台需重写 |
| CPU 占用 | ❌ 高，逐位翻转靠 CPU 循环 | ✅ 低，硬件移位寄存器自动完成，可配合 DMA |
| 速度 | ❌ 慢，受限于 CPU 执行频率 | ✅ 快，可达几十 MHz |
| 稳定性 | ❌ 中断打断会导致时序错误 | ✅ 硬件保证时序，不受中断影响 |
| 多设备 | ✅ CS 引脚随意分配 | ⚠️ SCK/MOSI/MISO 共用固定引脚，CS 可用任意 GPIO |
| 调试难度 | ✅ 逻辑清晰，易排查 | ❌ HAL 封装深，出错不好定位 |

**选用建议**：高速传输（Flash、LCD 刷屏）→ 必须用硬件 SPI；引脚紧张或非标准引脚挂设备 → 软件 SPI；学习 CPOL/CPHA → 先写一遍软件 SPI。

### HAL CAN — CAN 总线
**文档**: [HAL_CAN.md](HAL_CAN.md)

工业级多节点差分总线，抗干扰能力强。

**核心特点**
- 两线制（CANH/CANL），理论最多 ~110 个节点共享一根总线
- 硬件自动仲裁：ID 越小优先级越高，多节点同时发送不会冲突
- 内置 CRC 检测和自动重传，比 UART 可靠得多
- 最高 1Mbps；总线两端各需要 120Ω 终端电阻
- 支持标准帧（11 位 ID）和扩展帧（29 位 ID），每帧最多 8 字节数据

**适用场景**
- 多板嵌入式系统（机器人各关节控制板互联）
- 工业控制（变频器、伺服驱动器通信）
- 需要替代多路 UART 节省布线的多节点系统

---

### 单总线（DHT11 / DHT22）

单总线是自定义单线双向通信协议，一根数据线完成所有收发，协议有严格时序要求。

**核心特点**
- 仅需 1 根数据线，接线极简
- 时序由软件 GPIO 模拟，对中断延迟敏感
- DHT11：温度精度 ±2°C，湿度精度 ±5%RH
- DHT22：精度更高，温度 ±0.5°C，湿度 ±2%RH

**适用场景**
- 温湿度采集（成本低、接线简单）
- 不需要高精度或高速采样的环境监测

## 模拟外设

### HAL ADC — 模数转换
**文档**: [HAL_ADC.md](HAL_ADC.md)

将模拟电压（0~3.3V）转换为 12 位数字值（0~4095）。

**核心特点**
- 12 位分辨率，最多 16 个外部通道；ADC 时钟最高 14MHz
- 支持单次、连续、扫描（多通道）三种转换模式
- 支持阻塞轮询、中断、DMA 三种读取方式
- 多通道同时采集必须开启扫描模式 + DMA，数据自动写入数组
- 首次使用前必须校准：`HAL_ADCEx_Calibration_Start`
- ADC 引脚必须配为 `GPIO_MODE_ANALOG`

**电压换算公式**

$$实际电压 = \frac{ADC读取数值}{4095} \times 3.3V$$

**何时选哪种方案**

| 场景 | 方案 |
|------|------|
| 偶发读一次值 | 单次阻塞（Start + Poll + GetValue） |
| 持续读单路 | 连续模式轮询 |
| 多路并行采集，不占 CPU | 扫描 + DMA 循环 |
| 精确定时采样（如音频） | 定时器触发 |
| 读片内温度 | 温度传感器通道，采样时间必须用 239.5 周期 |

### HAL DAC — 数模转换
**文档**: [HAL_DAC.md](HAL_DAC.md)

将数字值转换为模拟电压输出，引脚固定为 PA4（通道 1）和 PA5（通道 2）。

**核心特点**
- 12 位分辨率，输出范围 0 ~ VDDA（通常 3.3V）
- 输出引脚必须配为 `GPIO_MODE_ANALOG`
- 支持软件触发、定时器触发和 DMA 连续输出
- 波形生成（正弦波、音频）必须用 DMA + 定时器触发，预先计算好数据数组

**适用场景**
- 输出固定偏置电压（如运放参考电压、传感器激励）
- 模拟量控制（调节电机转速、电动阀门开度）
- 音频播放（PCM 数据流，DMA + 定时器精确触发）
- 测试信号源（正弦波、锯齿波）

**DAC vs PWM + 低通滤波**

| 特性 | DAC | PWM + 滤波 |
|------|-----|-----------|
| 输出方式 | 直接输出平稳电压 | 高低电平快速通断，靠平均值调压 |
| 波形质量 | 平滑连续，无纹波 | 需滤波电路，仍有纹波 |
| 成本 | 需要 DAC 引脚（F1 仅 2 路）| 成本低，PWM 通道多 |
| 典型用途 | 音频输出、精准电压控制 | 电机调速、LED 调光 |

## 定时器与看门狗

### HAL TIM — 定时器
**文档**: [HAL_TIM.md](HAL_TIM.md)

STM32F1 中功能最丰富的外设，除了定时还承担 PWM、输入捕获、编码器等多种角色。

**定时器分类**

| 类型 | 实例 | 特点 |
|------|------|------|
| 高级定时器 | TIM1, TIM8 | 互补 PWM + 死区控制，用于 H 桥电机驱动 |
| 通用定时器 | TIM2~TIM5 | 4 通道，PWM / 输入捕获 / 编码器 |
| 基础定时器 | TIM6, TIM7 | 仅定时，无通道，资源占用最小 |

**各功能适用场景**

| 功能 | 典型应用 |
|------|---------|
| 基础定时中断 | 1ms / 10ms 系统心跳，周期任务调度 |
| PWM 输出 | 舵机（50Hz）、电机调速（1~20kHz）、LED 调光 |
| 输入捕获 | 超声波测距、测量信号频率/脉宽 |
| 编码器接口 | 旋转编码器计数，比 EXTI 方案更精确、不丢脉冲 |
| 定时器触发 ADC/DAC | 精确采样率，替代软件延时 |
| 互补 PWM + 死区 | H 桥/半桥电机驱动（防上下管同时导通） |

**PWM 关键参数**
- **频率（Hz）**：控制运行平顺度、LED 闪烁感（舵机 50Hz，电机 1~20kHz）
- **占空比（%）**：控制转速、亮度、功率大小

**输入捕获 vs 编码器模式**

| 特性 | 输入捕获 | 编码器模式 |
|------|----------|-----------|
| 自动识别方向 | ❌ 需自己写逻辑 | ✅ 硬件自动判断 |
| 自动计数增减 | ❌ | ✅ |
| 接线通道 | 单路 | TIM_CH1 + CH2 接 A/B 相 |
| 首选场景 | 测频率 / 脉宽 / 超声波 | 旋转编码器 / 电机测速 |

### HAL RTC — 实时时钟
**文档**: [HAL_RTC.md](HAL_RTC.md)

低功耗运行的时钟模块，在主电源断开时可由纽扣电池维持运行。

**核心特点**
- 独立运行，即使进入待机模式也能继续计时
- 时钟源：LSE（外接 32.768kHz 晶振，精度高）或 LSI（内部 RC，无需外部器件）
- 备份寄存器（20 字节）在断电后保持数据，可用魔数检测避免重复初始化
- 支持闹钟中断，到时间自动唤醒

**适用场景**
- 数据记录仪（每条数据附时间戳）
- 定时唤醒（低功耗设备定时采集、上报数据）
- 实时时钟显示
- 设备运行时长统计

**时钟源选择**：正式产品用 LSE（误差约 1 分钟/月）；原型验证或成本敏感场景用 LSI（误差约 3 分钟/天）。使用 LSE 需要 VBAT 引脚接纽扣电池。

---

### HAL IWDG — 独立看门狗
**文档**: [HAL_IWDG.md](HAL_IWDG.md)

基于独立 LSI 时钟的硬件看门狗，一旦启动无法软件关闭。

**核心特点**
- 使用独立 LSI 时钟，主时钟异常或停止模式下仍继续运行
- 超时时间 0.1ms ~ 26s，可配置
- 只检测"太晚喂狗"（程序死掉），不检测"太早喂狗"
- 调试时必须加 `__HAL_DBGMCU_FREEZE_IWDG()`，否则单步时会触发复位

**适用场景**
- 所有正式产品的防死机保护（无人值守设备首选）
- 通信超时保护（等待超时后自动复位重新初始化）

---

### HAL WWDG — 窗口看门狗
**文档**: [HAL_WWDG.md](HAL_WWDG.md)

基于 APB1 时钟的看门狗，有时间窗口限制——太早或太晚喂狗都会触发复位。

**核心特点**
- 只有在计数器值落入窗口区间内喂狗才有效
- 使用 APB1 时钟，进入停止/待机模式后停止运行
- 超时时间较短（毫秒级）
- 可配置早期唤醒中断（计数器接近复位前触发），给最后一次喂狗机会

**IWDG vs WWDG 选择**

| 需求 | 选择 |
|------|------|
| 防止系统完全死机 | IWDG（独立时钟，更可靠） |
| 检测程序时序异常（某任务执行时间不对） | WWDG |
| 最高可靠性要求 | 两者同时使用 |

## 存储与时钟

### HAL FLASH — 内部 Flash
**文档**: [HAL_FLASH.md](HAL_FLASH.md)

操作片内 Flash 存储器，用于保存配置参数和数据。

**核心特点**
- Flash 只能从 1 写成 0，写之前必须先按页擦除（每页 1KB，擦后全为 0xFF）
- 编程单位为半字（16 位），地址必须 2 字节对齐
- 擦写寿命约 10,000 次（远低于外部 EEPROM 的 100 万次）
- 操作前必须解锁，操作后立即锁定，防止意外写入

**适用场景**
- 保存设备配置参数（写入频率极低，每次开机读取）
- Bootloader 固件升级（接收新固件写入 Flash 后跳转）
- 生产时烧录出厂信息（只读）

**不适合的场景**：频繁写入的运行数据（如传感器日志）→ 用外部 EEPROM 或 SPI Flash。地址规划建议：程序从 Flash 头部开始，用户数据从末尾分配，两者互不重叠。

**Flash vs SRAM 对比**

| 特性 | Flash | SRAM |
|------|-------|------|
| 类比 | 硬盘 | 运行内存 |
| 存储内容 | 程序代码、常量、配置参数 | 变量、临时数据、堆栈 |
| 断电保留 | ✅ 保留 | ❌ 清空 |
| 读取速度 | 快 | 最快 |
| 擦写速度 | 慢（有次数限制） | 任意读写，无限制 |
| STM32F103C8T6 容量 | 64KB | 20KB |

## 系统与低功耗

### HAL PWR — 电源管理
**文档**: [HAL_PWR.md](HAL_PWR.md)

控制芯片的低功耗模式，适合电池供电设备延长续航。

**三种低功耗模式对比**

| 模式 | 功耗 | 唤醒时间 | 适用场景 |
|------|------|----------|---------|
| 睡眠（Sleep） | ~15mA | 立即 | 外设（DMA/ADC）需继续运行，CPU 等待中断 |
| 停止（Stop） | ~2μA | 几 μs | 长时间等待，外设可暂停，唤醒后需重配 PLL |
| 待机（Standby） | ~1μA | 几 ms（等同复位） | 极低功耗，唤醒后重新执行 main |

**关键注意点**
- 停止模式唤醒后系统时钟回退到 HSI（8MHz），必须重新调用 `SystemClock_Config()` 配置 PLL
- 待机模式唤醒相当于复位，代码从头执行，通过检查 `PWR_FLAG_SB` 标志判断唤醒来源
- PVD（电源电压检测）：电池供电设备在电压跌破阈值时触发中断，触发后应立即保存数据

**唤醒源**：睡眠模式任意中断可唤醒；停止模式需要 EXTI 或 RTC 闹钟；待机模式只能靠 WKUP 引脚（PA0）或 RTC 闹钟。

## 辅助模块

### HAL DMA — 直接内存访问
**文档**: [HAL_DMA.md](HAL_DMA.md)

在外设和内存之间直接搬运数据，无需 CPU 介入。

**三种搬运方向**

| 方向 | 说明 | 典型用途 |
|------|------|---------|
| 外设 → 内存 | 收数据 | ADC 采集结果存入数组、串口接收 |
| 内存 → 外设 | 发数据 | 串口批量发送缓冲区、SPI 刷屏 |
| 内存 → 内存 | 纯拷贝 | 数据备份、缓冲区搬移 |

**核心特点**
- DMA 通道分配固定（如 USART1_TX → DMA1_Channel4），不能任意选择
- 两种模式：单次（Normal，传完停止）和循环（Circular，自动重载，适合连续流）
- 通常不单独使用，而是作为 UART / ADC / SPI / I2C 的数据搬运后端

**何时必须用 DMA**
- UART 收发大量数据（>100 字节），避免频繁中断
- ADC 多通道连续采集，CPU 来不及逐个读取
- SPI 刷大屏（大量像素数据），刷屏期间 CPU 可处理其他事
- DAC 连续波形输出（音频、正弦波），需精确时序

**常用组合搭配**

| 组合 | 搬运方向 | 用途 |
|------|----------|------|
| ADC 多通道 + DMA | 外设 → 内存 | 持续采集多路模拟信号 |
| UART 大量收发 + DMA | 双向 | 高速串口通信 |
| SPI LCD 刷屏 + DMA | 内存 → 外设 | 快速刷新显示缓冲区 |
| DAC 波形输出 + DMA | 内存 → 外设 | 音频播放、正弦波生成 |

---

### HAL BKP — 备份寄存器
**文档**: [HAL_BKP.md](HAL_BKP.md)

位于后备域（Backup Domain）的掉电保持寄存器，与 RTC 共用 VBAT 供电。

**核心特点**
- STM32F103C8T6 共 10 个 16 位寄存器（BKP_DR1 ~ BKP_DR10），共 20 字节
- 只要 VBAT 有电（纽扣电池），数据永久保留；系统复位、看门狗复位均不清零
- 写入前必须解除写保护：`HAL_PWR_EnableBkUpAccess()`
- TAMPER 引脚可配置为入侵检测，触发后自动清除所有 BKP 数据

**BKP 与 RTC 配合使用**

BKP 和 RTC 同在后备域，天然是黄金搭档。经典用法：用 BKP 存一个标志位，上电时先检查标志，避免 RTC 时间被反复重置。

```
首次上电 → BKP 无标志 → 初始化 RTC，设置初始时间 → 写入 BKP 标志
再次上电 → BKP 有标志 → 跳过 RTC 设置 → RTC 继续走之前的时间
```

**与其他存储对比**

| 存储类型 | 断电保持 | 容量 | 擦写次数 | 典型用途 |
|----------|----------|------|----------|----------|
| BKP 寄存器 | ✅（需 VBAT） | 20 字节 | 无限制 | 少量关键参数、RTC 初始化标志 |
| 内部 Flash | ✅ 无需电池 | 64KB | ~10,000 次 | 程序代码、配置参数 |
| SRAM | ❌ 断电清零 | 20KB | 无限制 | 运行变量、堆栈 |

**适用场景**
- 上电标志（判断是否首次启动，避免重复初始化 RTC）
- 少量需掉电保持的配置参数（设备 ID、运行模式）
- 设备开机计数、运行次数统计
- 入侵检测（配合 TAMPER 引脚检测是否被非法拆机）

---

### HAL CRC — 硬件 CRC 校验
**文档**: [HAL_CRC.md](HAL_CRC.md)

硬件加速的 CRC-32 计算单元，每个 32 位字仅需 1 个时钟周期。

**核心特点**
- 固定使用 CRC-32 多项式（0x04C11DB7）
- 输入必须是 `uint32_t` 数组，字节数不是 4 的倍数时需手动补零
- 支持分段累加计算（`HAL_CRC_Accumulate`），适合流式数据
- 不支持 CRC-16（如 MODBUS），MODBUS 场景必须用软件实现

**适用场景**
- Flash 存储数据完整性校验（读出数据后验证）
- Bootloader 固件升级后校验固件完整性
- 通信帧附加校验值（大块数据，速度要求高）

## 操作系统

嵌入式 RTOS 适用于需要并发任务调度、有多个独立功能模块的复杂项目。

| 系统 | 特点 | 适用场景 |
|------|------|----------|
| FreeRTOS | 轻量级，任务调度、信号量、队列、消息机制完备；STM32CubeMX 直接集成；ESP-IDF 内核本身即 FreeRTOS | STM32 和 ESP32 均可，大多数项目首选 |
| RT-Thread | 国产 RTOS，生态丰富，支持组件化（文件系统、网络协议栈等）；STM32 和 ESP32 均支持 | 中大型嵌入式项目，需要丰富中间件 |

> **FreeRTOS 与 ESP32（IDF）：** ESP-IDF 内核本身就是 FreeRTOS，`#include "freertos/FreeRTOS.h"` 直接可用，无需额外集成。
>
> **RT-Thread 与 ESP32（IDF）：** 通过 IDF Component Manager 引入 `rt-thread` 组件即可使用，API 与 STM32 版本完全一致。

**FreeRTOS 完整使用指南**：[FreeRTOS_STM32_指南.md](FreeRTOS_STM32_指南.md)

涵盖 CubeMX 配置步骤、任务/队列/信号量/互斥量/事件组/任务通知/软件定时器的完整 API 用法、常见陷阱与典型工程结构。

**引入 RTOS 的时机**：任务超过 3~4 个且有优先级差异、需要任务间同步/通信、裸机调度逻辑变得复杂时，考虑引入 FreeRTOS。注意：CORTEX 模块中需按 FreeRTOS 要求设置内核中断优先级下限。

## 触控屏

触控屏通常由显示和触摸两个子系统构成，通过不同总线分别连接。

- **显示数据**：通过 **SPI 或并行总线** 高速传输，推荐硬件 SPI + DMA 刷屏
- **触摸坐标**：通常通过 **I2C** 回传坐标数据（如 FT6206、GT911 触摸控制器）
- **GUI 框架**：搭配 LVGL 等开源 GUI 框架可大幅简化界面开发

## 模块选型速查

### 按需求找模块

| 需求 | 模块 |
|------|------|
| 控制 LED / 继电器 | GPIO（推挽输出） |
| 读取按键 | GPIO（输入 + 上拉）|
| 按键触发中断 | GPIO + EXTI |
| 调试打印 / printf | UART（阻塞，fputc 重定向） |
| 与传感器通信（低速，多设备） | I2C |
| 与 Flash / 屏幕通信（高速） | SPI |
| 多板系统，长距离抗干扰通信 | CAN |
| 采集模拟传感器值 | ADC |
| 输出模拟控制信号 | DAC |
| 生成 PWM 控制电机 / 舵机 | TIM（PWM 输出） |
| 系统心跳 / 周期任务 | TIM（基础定时中断） |
| 测量脉宽 / 频率 | TIM（输入捕获） |
| 旋转编码器 | TIM（编码器接口）|
| 记录时间 / 定时唤醒 | RTC |
| 防程序死机 | IWDG |
| 检测程序时序异常 | WWDG |
| 保存配置参数（很少写） | FLASH |
| 大量存储 / 频繁写入 | 外部 SPI Flash / EEPROM |
| 掉电保持少量关键参数 | BKP（需 VBAT 供电） |
| 电池供电省电 | PWR（低功耗模式）|
| 数据校验 | CRC |
| CPU 卸载数据搬运 | DMA（配合各外设使用） |
| 温湿度采集（低成本） | DHT11 单总线 |
| 多任务并发调度 | FreeRTOS / RT-Thread |
| 触摸屏显示 | SPI（显示）+ I2C（触摸）+ LVGL |

### 通信协议对比

| 协议 | 线数 | 最高速度 | 多节点 | 抗干扰 | 典型应用 |
|------|------|---------|--------|--------|---------|
| UART | 2 | ~10Mbps | 不直接支持 | 一般 | 调试、模块通信 |
| I²C | 2 | 400kHz | ✅ 地址区分 | 弱（需短距离）| 传感器、EEPROM |
| SPI | 4+ | 18Mbps（F1）| ✅ 靠 CS 线 | 中 | Flash、屏幕、ADC |
| CAN | 2 | 1Mbps | ✅ 硬件仲裁 | 强 | 工业、多板互联 |
| 单总线 | 1 | 低 | 不支持 | 弱 | DHT11 温湿度 |

### 存储方案对比

| 方案 | 擦写寿命 | 容量 | 写入速度 | 适用 |
|------|---------|------|---------|------|
| 内部 Flash | ~10,000 次 | 受限 | 慢（需擦除） | 配置参数，极少写入 |
| BKP 寄存器 | 无限制 | 20 字节 | 快 | 掉电保持的极少量参数 |
| AT24C EEPROM | ~1,000,000 次 | 256B~64KB | 中（5ms/写） | 频繁更新的小数据 |
| W25Q SPI Flash | ~100,000 次 | 1MB~128MB | 快 | 固件存储、大量数据 |
