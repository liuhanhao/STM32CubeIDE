# SquareLine Studio — 导出与集成

## 一、导出前检查

导出前确认以下项目：

- [ ] 所有屏幕名称、控件名称已按规范命名
- [ ] 分辨率与实际屏幕一致
- [ ] 颜色深度与 `lv_conf.h` 中 `LV_COLOR_DEPTH` 一致
- [ ] LVGL 版本与工程中使用的版本一致
- [ ] 自定义事件函数名已确定（避免后期改名）

---

## 二、导出操作

**方式一：** 顶部菜单 **Export → Export UI Files**
**方式二：** 顶部工具栏 `Cmd + E`

设置导出路径（建议直接导出到工程的 `main/ui/` 目录），点击 **Export**。

---

## 三、导出文件结构

```
导出目录/
├── ui.c                    ← 主入口，调用各屏幕初始化
├── ui.h                    ← 所有控件变量的声明（extern）
├── ui_events.c             ← 自定义事件函数（填业务逻辑）
├── ui_events.h
├── ui_helpers.c            ← 内部辅助函数（屏幕切换动画等）
├── ui_helpers.h
├── ui_comp.c               ← 组件代码（如有组件）
├── ui_comp.h
├── screens/
│   ├── ui_Screen_Home.c    ← 每个屏幕一个文件
│   ├── ui_Screen_Home.h
│   ├── ui_Screen_Setting.c
│   └── ui_Screen_Setting.h
├── fonts/
│   └── lv_font_cn_16.c    ← 导入的字体文件
└── images/
    ├── ui_img_logo.c       ← C 数组格式图片（Binary 格式则为 .bin 文件）
    └── ui_img_icon_wifi.c
```

### 哪些文件不能手动修改

| 文件 | 是否可手动修改 | 说明 |
|------|-------------|------|
| `ui_Screen_xxx.c/h` | ❌ 不可修改 | 每次导出覆盖 |
| `ui.c / ui.h` | ❌ 不可修改 | 每次导出覆盖 |
| `ui_helpers.c/h` | ❌ 不可修改 | 每次导出覆盖 |
| **`ui_events.c`** | ✅ 在这里写 | **导出不覆盖** |
| **`ui_events.h`** | ✅ 可添加声明 | 导出不覆盖 |

## 四、集成到 ESP32（ESP-IDF + PlatformIO）

### 4.1 目录结构

将导出的文件放入工程的 `main/ui/` 目录：

```
esp32_project/
├── main/
│   ├── CMakeLists.txt
│   ├── main.c
│   ├── display.c / display.h
│   ├── touch.c / touch.h
│   └── ui/                         ← SquareLine 导出目录
│       ├── ui.c / ui.h
│       ├── ui_events.c / ui_events.h
│       ├── ui_helpers.c / ui_helpers.h
│       ├── screens/
│       │   ├── ui_Screen_Home.c
│       │   └── ui_Screen_Setting.c
│       ├── fonts/
│       │   └── lv_font_cn_16.c
│       └── images/
│           └── ui_img_logo.c
├── CMakeLists.txt
└── platformio.ini
```

### 4.2 CMakeLists.txt

```cmake
# 收集 ui/ 目录下所有 .c 文件（递归）
file(GLOB_RECURSE UI_SOURCES "ui/*.c")

idf_component_register(
    SRCS
        "main.c"
        "display.c"
        "touch.c"
        ${UI_SOURCES}          # ← 包含所有 ui 文件
    INCLUDE_DIRS
        "."
        "ui"
        "ui/screens"
        "ui/fonts"
        "ui/images"
    REQUIRES
        esp_lcd
        esp_timer
        freertos
        driver
)
```

### 4.3 main.c 中调用

```c
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "freertos/semphr.h"
#include "lvgl.h"
#include "display.h"
#include "touch.h"
#include "ui/ui.h"      // ← 包含 SquareLine 导出的头文件

SemaphoreHandle_t lvgl_mutex;

static void gui_task(void *arg)
{
    display_init();
    touch_init();
    lvgl_tick_init();

    ui_init();          // ← 调用 SquareLine 生成的 UI 初始化函数

    while (1) {
        if (xSemaphoreTake(lvgl_mutex, pdMS_TO_TICKS(10))) {
            lv_timer_handler();
            xSemaphoreGive(lvgl_mutex);
        }
        vTaskDelay(pdMS_TO_TICKS(5));
    }
}

void app_main(void)
{
    lvgl_mutex = xSemaphoreCreateMutex();
    xTaskCreatePinnedToCore(gui_task, "gui", 8192, NULL, 5, NULL, 1);
}
```

## 五、集成到 STM32（STM32CubeIDE / PlatformIO）

### 5.1 目录结构

```
stm32_project/
├── Core/
│   ├── Src/
│   │   ├── main.c
│   │   └── ...
│   └── Inc/
└── UI/                         ← SquareLine 导出目录
    ├── ui.c / ui.h
    ├── ui_events.c / ui_events.h
    ├── screens/
    ├── fonts/
    └── images/
```

### 5.2 添加文件到 STM32CubeIDE

1. 右键项目 → **New → Folder**，创建 `UI` 文件夹
2. 将导出文件复制进去
3. 右键 `UI` 文件夹 → **Properties → C/C++ Build**，确认包含在构建中
4. 项目属性 → **C/C++ General → Paths and Symbols** → 添加 Include 路径：
   - `UI`
   - `UI/screens`
   - `UI/fonts`

### 5.3 main.c 中调用（STM32 HAL）

```c
#include "lvgl.h"
#include "ui.h"       // SquareLine 导出

/* 在 LVGL 和显示驱动初始化完成后 */
void MX_LVGL_Init(void)
{
    lv_init();
    /* ... 注册显示驱动和触摸驱动 ... */
    ui_init();        // ← 调用 SquareLine UI 初始化
}

/* 主循环 */
while (1)
{
    lv_timer_handler();
    HAL_Delay(5);
}
```

## 六、业务逻辑与 UI 联动

所有业务代码写在 `ui_events.c` 中，通过 LVGL API 更新 UI 控件：

```c
/* ui_events.c */
#include "ui.h"

/* 传感器数据更新 UI（在业务任务中调用，注意加锁）*/
void update_sensor_ui(float temperature, float humidity)
{
    lv_label_set_text_fmt(ui_Label_Temperature, "%.1f °C", temperature);
    lv_label_set_text_fmt(ui_Label_Humidity, "%.0f %%", humidity);

    /* 更新进度条 */
    lv_bar_set_value(ui_Bar_Humidity, (int32_t)humidity, LV_ANIM_ON);
}

/* 按钮事件（SquareLine 调用）*/
void btn_start_handler(lv_event_t *e)
{
    lv_event_code_t code = lv_event_get_code(e);
    if (code == LV_EVENT_CLICKED) {
        start_measurement();
        lv_label_set_text(ui_Label_Status, "测量中...");
    }
}
```

## 七、重新导出的注意事项

每次在 SquareLine Studio 修改设计后重新导出：

1. **`ui_events.c` 不会被覆盖**，你的业务逻辑安全
2. **其他所有 `.c/.h` 文件都会被覆盖**，不要手动改它们
3. 如果添加了新的屏幕或控件，`ui.h` 会更新，新的控件变量会自动声明
4. 导出路径保持不变，直接覆盖即可，无需修改 CMakeLists.txt

## 八、完整开发工作流总结

```
1. 新建 SquareLine 项目（设置分辨率、颜色深度、LVGL 版本）
        ↓
2. 设计 UI（拖控件、设样式、绑事件）
        ↓
3. 预览测试（Run 按钮）
        ↓
4. 导出 UI 文件（Export → 导出到工程 ui/ 目录）
        ↓
5. 在 ui_events.c 中填入业务逻辑
        ↓
6. 编译烧录到硬件测试
        ↓
7. 回到步骤 2 迭代修改（只有 ui_events.c 需要保留）
```
