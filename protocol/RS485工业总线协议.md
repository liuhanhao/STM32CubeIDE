# RS-485 工业总线协议

## 概述

RS-485 是一种**差分信号、半双工**的串行通讯标准（由 EIA/TIA 制定）。它在 UART 的基础上增加了差分驱动，使得传输距离更远、抗干扰能力更强，是工业现场最广泛使用的有线通讯标准。

- 通讯方式：异步、半双工（同一时刻只能一个方向传输）
- 所需引脚：A（+）、B（-）两根差分线
- 传输距离：最远 **1200 米**（低速时）
- 节点数量：最多 **32 个**标准节点（使用增强型收发器可达256个）
- 典型速度：9600 bps ～ 10 Mbps（速度越高，距离越短）

---

## 差分信号原理

```
逻辑1（Mark）：A - B > +200mV
  A线: ‾‾‾‾‾‾‾‾‾‾‾‾  约 +3.5V
  B线: ____________  约 -3.5V（或接近GND）

逻辑0（Space）：A - B < -200mV
  A线: ____________  约 -3.5V
  B线: ‾‾‾‾‾‾‾‾‾‾‾‾  约 +3.5V
```

> 差分信号的优势：即使线缆上有几伏的共模干扰，A-B 的差值仍然正确，接收方不受影响。

## 硬件连接

### 半双工（2线制，最常用）

```
主机（MCU）                    从机1              从机2
  UART_TX ─→ DE/RE ─→ 收发器                    
  UART_RX ←─         收发器
                        │A(+)
                        ├──────────────────────────
                        │B(-)
                        ├──────────────────────────
                       120Ω                      120Ω
                      终端电阻                  终端电阻
```

### 全双工（4线制）

```
主机                    从机
  TX+ ──────────────→  RX+
  TX- ──────────────→  RX-
  RX+ ←──────────────  TX+
  RX- ←──────────────  TX-
```

## 方向控制（DE/RE 引脚）

RS-485 收发器有方向控制引脚：

| 引脚 | 功能 | 说明 |
|------|------|------|
| DE | Driver Enable | 高电平使能发送 |
| RE | Receiver Enable | 低电平使能接收（通常与DE反相连接） |

```c
// 切换到发送模式
void RS485_SetTX(void) {
    GPIO_SetHigh(DE_PIN);   // 使能发送驱动
    GPIO_SetHigh(RE_PIN);   // 禁用接收（RE高=禁用）
}

// 切换到接收模式
void RS485_SetRX(void) {
    GPIO_SetLow(DE_PIN);    // 禁用发送驱动
    GPIO_SetLow(RE_PIN);    // 使能接收（RE低=使能）
}
```

## Modbus RTU 协议（RS-485 最常用的上层协议）

RS-485 只定义物理层，实际应用中通常配合 **Modbus RTU** 协议使用。

### Modbus RTU 帧格式

```
┌──────────┬──────────┬──────────┬──────────────────┬──────────┐
│ 设备地址  │ 功能码   │  数据    │      数据        │  CRC     │
│  1字节   │  1字节   │  N字节   │                  │  2字节   │
└──────────┴──────────┴──────────┴──────────────────┴──────────┘
```

### 常用功能码

| 功能码 | 说明 |
|--------|------|
| 0x01 | 读线圈（读取数字输出状态） |
| 0x02 | 读离散输入（读取数字输入状态） |
| 0x03 | 读保持寄存器（最常用，读取数据） |
| 0x04 | 读输入寄存器 |
| 0x06 | 写单个寄存器 |
| 0x10 | 写多个寄存器 |

## 完整通讯示例

**场景：** 主机通过 RS-485 + Modbus RTU 读取温度传感器（地址 0x01）的温度值

### 读取保持寄存器（功能码 0x03）

**主机发送请求：**

```
字节:  [01]  [03]  [00 01]  [00 01]  [D5 CA]
        ↑     ↑      ↑         ↑        ↑
     设备地址 功能码 起始寄存器  读取数量   CRC校验
              读寄存器 (0x0001)  (1个寄存器)
```

**从机（传感器）返回响应：**

```
字节:  [01]  [03]  [02]  [01 0E]  [B8 53]
        ↑     ↑     ↑      ↑         ↑
     设备地址 功能码 字节数  数据值    CRC校验
                    (2字节) (0x010E=270 → 27.0°C)
```

### 完整时序过程

```
1. 主机切换到发送模式（DE=1）
2. 主机发送请求帧：01 03 00 01 00 01 D5 CA
3. 主机等待发送完成（等待最后一个字节发出）
4. 主机切换到接收模式（DE=0，RE=0）
5. 等待从机响应（超时时间通常 100ms）
6. 从机接收到请求，解析地址=0x01（是自己），功能码=0x03
7. 从机读取寄存器 0x0001 的值 = 270（27.0°C）
8. 从机切换到发送模式，发送响应：01 03 02 01 0E B8 53
9. 从机切换回接收模式
10. 主机接收响应，解析温度值 = 270 → 27.0°C
```

### 代码示例

```c
// Modbus RTU 读保持寄存器
uint16_t Modbus_ReadRegister(uint8_t slaveAddr, uint16_t regAddr) {
    uint8_t request[8];
    uint8_t response[7];

    // 构建请求帧
    request[0] = slaveAddr;           // 设备地址
    request[1] = 0x03;                // 功能码：读保持寄存器
    request[2] = (regAddr >> 8) & 0xFF; // 寄存器地址高字节
    request[3] = regAddr & 0xFF;        // 寄存器地址低字节
    request[4] = 0x00;                // 读取数量高字节
    request[5] = 0x01;                // 读取数量低字节（1个寄存器）
    uint16_t crc = CRC16_Modbus(request, 6);
    request[6] = crc & 0xFF;          // CRC低字节
    request[7] = (crc >> 8) & 0xFF;   // CRC高字节

    // 发送请求
    RS485_SetTX();
    UART_Send(request, 8);
    UART_WaitTxComplete();
    RS485_SetRX();

    // 接收响应（超时100ms）
    if (UART_Receive(response, 7, 100) != 7) {
        return 0xFFFF;  // 超时错误
    }

    // 解析响应
    if (response[0] == slaveAddr && response[1] == 0x03) {
        return (response[3] << 8) | response[4];  // 返回寄存器值
    }
    return 0xFFFF;  // 错误
}
```

## 多节点通讯示例

**场景：** 1个主机轮询3个从机（地址1、2、3）的温度

```
时间轴：
t=0ms:   主机 → 从机1：01 03 00 01 00 01 [CRC]  （查询从机1）
t=5ms:   从机1 → 主机：01 03 02 01 0E [CRC]      （27.0°C）
t=10ms:  主机 → 从机2：02 03 00 01 00 01 [CRC]  （查询从机2）
t=15ms:  从机2 → 主机：02 03 02 01 1C [CRC]      （28.4°C）
t=20ms:  主机 → 从机3：03 03 00 01 00 01 [CRC]  （查询从机3）
t=25ms:  从机3 → 主机：03 03 02 00 F5 [CRC]      （24.5°C）
t=30ms:  循环重新开始...
```

> 注意：同一时刻只有一个设备在发送，其他设备处于接收监听状态。

## 常见问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 通讯不稳定 | 缺少终端电阻 | 在总线两端各加120Ω |
| 数据错误 | 方向切换时序问题 | 发送完成后延迟再切换到接收 |
| 远端设备无响应 | 总线悬空 | 加偏置电阻（A接上拉，B接下拉） |
| 多设备冲突 | 同时发送 | 确保主从架构，从机只在被查询时响应 |

## 典型应用

- 工业传感器网络（温度、压力、流量）
- PLC 与变频器通讯
- 楼宇自动化（门禁、照明控制）
- 电能表抄表系统
- Modbus 协议底层传输
